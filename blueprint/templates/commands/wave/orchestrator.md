# Wave Orchestrator — Dual-Chat Wave Runner

$ARGUMENTS

This chat is the **wave_orchestrator** — stages, briefs, verifies every task, gates the merge, closes out; the **builder** (first argument = its tmux name from `/chat:ls`) implements. Judgment never delegates down; implementation never re-decides the spec. First arg `lanes` (launch) or `resume-lanes` (re-entry) → § Lanes — multi-builder trains; a builder-chat name runs the single-lane protocol below unchanged.

**Autonomous execution contract:** pre-flight (O0) is the only gate; once it passes, the wave runs to completion. Founder present → relay product-level SPEC-CONFLICTs; founder unreachable (God speed) → rule by best judgment on the reconciled spec, or mark that task BLOCKED-DEFERRED and continue disjoint tasks. **God speed's ONLY failure is waiting for the founder:** every founder-only question is FORECAST at O0 and closed before launch; a wave that stops mid-flight for a founder answer has failed — reroute through protocol, rule, or BLOCKED-DEFER, and log the reversal retro.

**Models:** founder-defined per chat; default **Opus orchestrator · Sonnet (xhigh effort) builder** — verify the builder pane's statusline model at every launch/respawn, correct via `/model` inject; a frontier-hands grant may flip it up per span, back after. Launch THIS chat with `ENABLE_PROMPT_CACHING_1H=1` — 1h cache, orchestrator-ONLY (write-heavy chats pay 2× writes); the waiter's 10m wake stays cache-warm under it.

## Resolve the task file

- **Empty/blank second arg:** task file is `wave.md` at repo root.
- **File path:** read that file. **Description:** parse as inline tasks (must still meet the ZERO-GAP bar).
- Extract `**Epic:** {name}` (`none` if absent; train — a partition's own `**Epic:**` header line overrides the top-level for that wave). Choose a kebab-case `{wave-name}` (2-4 words); verify uniqueness against `$WAVES/`, `docs/dev/builds/`, `tmp/dev/archive/{builds,waves}/` — collision → `-v2` or a more specific name.
- **Wave train:** tasks spanning more than one code impact area never run as one wave — the spec's `## Wave {k}:` headings + their header blocks (**Changes / Touches / Depends / Tasks** — `/wave:refine` § wave.md format) declare the partitions: build the train map by grepping the headings + the four `**Field:**` lines beneath each and split on heading line numbers mechanically, NEVER by reading the full source. Undeclared multi-area file → derive the partition map ONCE — a Sonnet reader returns per-task area + file-plan cards AND surfaces every preamble/rules block with the task range it binds — then write the `## Wave` headings + header blocks into the source before extracting, so every later wave and resume reads the same ruling; mid-train you may subdivide a partition that proves too big, never merge or reshuffle across declared boundaries. Founder override: remaining partitions may merge into ONE wave cycle — one worktree, one BRIEF spanning their still-READ-ONLY manifests, one ceremony. Each partition is a full wave (own worktree, own O0→O6 cycle, merged to `main`) before the next starts. This invocation runs the FIRST remaining partition; § O6 chains the rest. Preamble rides by scope: the `**Epic:**` header + sections binding ALL tasks go into every partition's manifest; a section binding a task subset (a later `# Tasks` block's preamble, a per-wave rules block) goes into exactly the partitions holding tasks it governs.

## O0 — Pre-flight

FAIL FAST before any setup. All fatals print a diagnostic and exit — no wave dir, no worktree.

1. **Singleton lock** — `mkdir tmp/wave-{wave-name}.orchestrator` (atomic) and write `$$ + timestamp` inside. Exists already → another orchestrator owns this wave: refuse, point at it. Released at the § O6 close (per-wave-name; a boundary holds two — closing + staged — per lane).
2. **ZERO-GAP shape** — every task carries the 10 refine sections (routing, data model, contracts, file plan, behaviors, boundaries, anchors, flow, why, summary) plus `**Build agents:**`. Loose prose → route to `/wave:refine`. **Train:** every `## Wave` heading must grep-recover all four header fields — a missing field is fatal (declared spec → back to `/wave:refine`; fallback → you wrote them, fix them).
3. **Existence + conflict checks** — grep every named entity (components, tables, endpoints, files); nonexistent referent or incompatible edits to one target across tasks → fatal. **Train:** scope items 2–3 to the current partition + preamble — an entity a prior partition creates resolves against merged `main` at that wave's pre-flight; cross-partition sequenced edits are ordering, not conflicts.
4. **Command triage** — `[CMD: /jc]` tasks run via `Skill("jc", ...)` now; `[CMD: /km]` via `Skill("km", ...)` BEFORE launch (builds may depend on the knowledge files). Pre-launch /km serves knowledge the build consumes from task 1; a task whose own proof needs PRE-CHANGE state (snapshot fixtures) marks its km INTERLEAVED — it fires at task time on the builder's `km-needed T{N}` ping. Any task whose file plan touches `{AI_PROJECT}/knowledge/` or `.claude/**` is split out here to /km//pcm — the builder never writes those paths, even when a gate marker is open; every km-coupled GO states it verbatim: "ALL knowledge/ writes are the orchestrator's — ping `km-needed` and hold; you never invoke /km yourself" (a present-artifact port is still the orchestrator's write). /km on WORKTREE knowledge files stamps the gate marker at the worktree's own `tmp/` (`$WT/tmp/professor_km_active` — km-guard anchors to the edited file's repo root; a worktree IS its own root); deletions follow km.md's checklist (file + REGISTRY.json entry + verify remaining paths resolve).
5. **Builder chat** — first argument names it; verify it exists and idles (`/chat:ls`). Missing → spawn `/chat:new --detach {wave-name}_builder` (own tmux socket = crash isolation). Record its name + your own (`/chat:whoami`) for the brief.
6. **Dirty main** — allowed; default `[CarryWIP: leave]`; gitter SETUP owns the mechanics.

## O1 — Stage

1. `mkdir -p $WAVES/{wave-name}`; copy the spec to `manifest.md`; **if the source is root `wave.md`, reset it to the `# Tasks` stub in the same atomic step**. **Train — the WHOLE-TRAIN SPLIT (first invocation only):** (a) gitter DOCS-COMMIT the pre-split source; (a2) pool trains: copy `## Priority` + `## Conflict graph` verbatim to `docs/dev/trains/{train-name}/pool.md` (`mkdir -p` the train dir when `/wave:schedule` didn't create it) — the dispatch source once the stub reset lands; (b) split EVERY partition into its own `$WAVES/{kebab-area}/manifest.md` — the `## Wave {k}:` heading + four header fields (+ its `**Lane:**` field when present) + a `**Train:** {source-path}` stamp line + its scoped preamble/rules blocks + its tasks, all line-range ops, never a full-file read; a manifest that already exists for a heading is SKIPPED, never rewritten; (c) verify the task-count sum across manifests against the source; (d) ONLY then reset the source to the stub. Manifests are READ-ONLY after the split — any later step that would rewrite one is a fatal diagnostic (exceptions: the WATCHER-approved rebalance `**Lane:**` re-stamp — § Idle capacity; and a clause that VIOLATES root `CLAUDE.md` MANDATORY law — a manifest instructing a builder to log {SENSITIVE_DATA}, ship {FORBIDDEN_DOMAIN_OUTPUTS}, or hardcode a secret is a root-law violation, not a spec to preserve: strike it in EVERY copy it was split into, since the clause propagates byte-identical and a fix to one leaves the others to re-teach it). Only the CURRENT wave gets item 2's STATE.md; later manifests stay STATE-less (inert to the wake discriminator and heartbeat) until § O6.3 stages them.
2. Seed `STATE.md` delta-structured: TOP = live resume brief + lifecycle stamp (`STAGED → CUT → LAUNCHED → BOUNDARY-ACTIVE → GREEN → CLOSED`, rewritten at each transition) + `## Next`; marker line `<!-- APPEND-ONLY BELOW — never rewrite -->`; below = locked decisions from refinement, then appended facts. Seed `report.md` skeleton (task table, gate slots, a `**Merge SHA:**` slot the walker's scout greps) and `retro.md` (header line only — the steering-conscience ledger, § O4). **Train:** after item 1's split the QUEUE is the set of Train-stamped manifests — a wave dir with STATE.md is current or done, one without is un-run; the source stub never holds partitions again.
3. **gitter SETUP** — `Pipeline: {wave-name}. Phase: SETUP. CarryWIP: leave.` One worktree `$WT`, branch `pipeline/{wave-name}`, ports + `.env.ports`, `ports.md`. SETUP also provisions the recurring gate-killers: copy gitignored `.env.local`s into the worktree, clear the {UI_FRAMEWORK} test cache, align `.env.test` ports to the allocated pipeline ports, regenerate FE codegen when the branch's generated files trail its BE schema (stale generated files dump accumulated drift into the first FE-codegen task's diff). **Train:** SETUP here is the first wave only — a boundary-staged wave defers it to § O6.4, so its worktree is cut only from verified `main`.
4. **Task map** — order the spec's tasks per its sequencing directive (a group = consecutive tasks). Mark **milestones**: the spec's declared milestone markers, else every group boundary, else every 4th task. Mark **`frontier-hands:`** on any span whose cost-of-being-wrong warrants a frontier-tier hand (novel algorithm, auth/consent path, concurrency, {DOMAIN_ADJ}-adjacent logic) — hand-tier judgment is the orchestrator's, never the builder's; everything unmarked runs Sonnet; stamp the map's frontier alias per `CLAUDE.md` § Model Selection (durable default `opus`). Write the map into STATE.md TOP.
5. **Waiter (the 10m wake)** — launch ONCE, train-wide, as YOUR OWN background Bash task (`run_in_background`): `.claude/scripts/wave-wait.sh` — it exits the moment an event lands in the spool `tmp/wave-sensor/events.log` (builder pings, watcher alarms) or after its max-wait (`HEARTBEAT-TIMEOUT`); its exit notification is this chat's guaranteed wake (§ O4). Relaunching it is the LAST act of every turn. **Launch it DIRECT — never a trailing `&` inside `run_in_background`:** double-backgrounding exits the wrapper at once, reparents the real waiter to PID 1, and the harness tracks nothing, so its exit notification wakes NOBODY — a deaf waiter that `kill -0` still calls alive. **Every launch ends with its tracking proof** — `ps -o ppid= -p $(cat tmp/wave-sensor/wait.pid)`; PPID 1 = reparented = deaf, kill and relaunch direct. **Before a known-long SYNCHRONOUS sub-agent (a gitter merge, a forensic), launch the waiter with a max-wait that outlives it — `wave-wait.sh 2400`:** a heartbeat exit landing while this loop is blocked wakes nobody and leaves the chat waiter-less until the call returns. Nothing is LOST when it happens (the cursor preserves every spooled event for the next waiter) — the cost is latency, not correctness, so do not over-engineer it.
6. **Founder-question forecast (gate)** — enumerate every founder-only item this wave will hit: secrets to place, live-pipeline/deploy reviews, destructive-op ratifications, merge nods, classifier-sensitive ops (the manifest's `WATCH:`/founder-ratified lines + `Touches` on live infra name them). CLOSE them all before launch — secrets PLACED and verified, ratifications materialized where they will be needed (a founder line in the executing session, or his permission rule), nods recorded pre-authorized in the manifest. An open founder question at launch is pre-flight-fatal; one discovered mid-flight is a reversal retro.

## O2 — Reconcile (replaces plan/architecture)

Fan out ONE read-only reconciler per routed project against `$WT` (`Explore`, Sonnet; parallel; boundary prep runs them against post-merge `main` — § O6.3). Each verifies its project's portion of the spec against live code and returns binding, reality-wins corrections:

- **Anchors + file plan** — every Named anchor, path, symbol, and signature the spec cites exists as described; count call sites the spec enumerates (an undercount ships a compile failure); a path cited as a DATA SOURCE must CONTAIN the claimed data, not merely exist; every to-DELETE file grep-checked single-purpose — kept symbols convert it to `EDIT (strip X)` by def-boundaries; a deleted shared symbol/field/state-key's coupling enumerated tree-wide — readers AND every test literal, including the typechecker-invisible reader classes: raw-SQL strings and {API_PROTOCOL} query/mutation DOCUMENTS (embedded operation strings, schema definition files, generated ops select dropped fields silently); under-cited scope = binding correction. Anchor verification confirms the anchor DOES what the spec claims (grep the symbol body/docstring) — existence-only checks pass on the wrong file; cited paths are ls/grep-confirmed VERBATIM, never manifest-copied — a path one directory short ships a failed first read.
- **Cross-system literals** — every literal the spec introduces that joins a live registry or convention (metric names, {QUEUE} message types, audit-map keys, enum values, i18n keys): grep the live convention; flag divergence.
- **Claims panel** — walker MANIFEST-VERIFY (`scriptPath:'.claude/workflows/wave-walker.js'`, `args.manifestPath`) probes every load-bearing claim refute-first — behavioral derivations, premises (anchor existence is not enough) — and flags cross-task conflicts + freeloader tasks; its REFUTED/PARTIAL verdicts and conflicts fold into the corrections; returned coverage must show every task mined (`droppedClaimIds` empty) — re-run with a higher `maxClaims` otherwise.

- **Behavioral triggers + proof mechanisms** — every condition the spec's behaviors state has a code path in the plan (enumerate the stated conditions; a None-only guard against a spec that also names empty is a gap); a task's cited proof mechanism (eval harness, gate) demonstrably COLLECTS the artifact the task changes — open the collector; a gate that cannot observe the changed output is not a gate; a behavior adding a NEW LLM call must name its validated prompt artifact or the correction stages it out (refine's RND-carrier gate owns the upstream half).
- **Coupling + principal deltas** — a task whose code drops/renames a `{slot}` a live prompt references is `[CMD:/km]`-coupled up front (persist-hop check both directions, never discovered mid-task as a KeyError); a wave promoting a role to a live authenticating principal runs a new-principal threat-model delta: enumerate every resolver fenced by {ORG_UNIT}-access alone and rule the new principal's reachability BEFORE merge (a {SENSITIVE_DATA} leak shipped exactly this way).
- **The wave's own SCOPE is a coverage claim — state it or it is a hole.** For a wave that cures a CLASS (every error-renders-as-absence; every {SENSITIVE_DATA}-in-a-log; every unguarded X), the reconcile enumerates EVERY reachable instance per surface and maps each to a task or an EXPLICIT defer. An instance nobody claims is not out of scope — it is an unstated hole, and it will survive the very wave written to kill it. (The honest-absence wave shipped with a reachable absence-render — a composite empty-state reached via a pipeline-step failure — that no task claimed: the disease inside the cure, at the scope altitude.)
- **Persist hops** — every new persisted field's write path AND read-back path named; a missing reader is a binding correction. The value must be IN SCOPE at the named write site — trace phase/graph ordering; a write that runs before its value is computed structurally cannot persist it.
- **CI/workflow inputs** — every operator- or remote-sourced value (workflow_dispatch inputs, box-written files) crossing into a git/shell sink carries fail-closed format validation in the mechanism itself; a brief introducing a new `${{ github.event.inputs.* }}` shows the hand the nearest `env:`-passing pattern in the same file as the required template — raw splicing into a script body is command injection.

Write the merged result to `$WAVES/{wave-name}/manifest-corrections.md`. The spec + corrections are the build truth; a correction contradicting founder intent goes through the SPEC-CONFLICT rule instead. Corrections are wave-scoped — verdicts and messages cite only THIS wave's correction IDs; concurrent lanes' reconciles never cross-cite.

## O3 — Brief & launch

**Task-class law:** planning, analysis, design, and advisory output are NEVER dispatched to a builder — product thinking lives upstream in `/wave:refine` ZERO-GAP specs, and residual thinking (reconciliation, rulings, probe shapes) is THIS chat's own work; every builder dispatch carries only exact build instructions — files, symbols, tests, verbatim spec pointers.

Write `$WAVES/{wave-name}/BRIEF.md` for the builder:

- **Identity & env** — worktree path, branch, ports, orchestrator chat name; test stack is builder-owned: `make -C $WT/{INFRA_PROJECT} nuke-test-pipeline PIPELINE={wave-name}` → `up-test-pipeline` → `db-setup-test-template-pipeline` → `db-setup-test-pipeline` → `health-test-pipeline`; stand-up and teardown are the builder's (§ O6.3 verify).
- **Build truth** — pointers to `manifest.md` + `manifest-corrections.md` + the task map. **A BRIEF quotes a manifest constraint VERBATIM or points at it — never paraphrases it looser.** A loose restatement invites a builder to satisfy the BRIEF and miss the spec: the manifest is the truth, the BRIEF is a pointer to it, and where they differ the manifest wins (a builder that chooses the manifest over the BRIEF is right, and the BRIEF is the defect). Implement tasks in order; the per-task `**Build agents:**` line names when to spawn db-admin (`{INFRA_PROJECT}/.claude/agents/db-admin.md`) or ui-ux (`{project-ui}/.claude/agents/ui-ux.md`); the code is written by the builder's Sonnet dev hands (its own turns coordinate and judge — wave/builder.md § Orchestrated mode).
- **Task loop + guard rails** — canonical home: `wave/builder.md` § Orchestrated mode (task loop, END-YOUR-TURN, checkpoint discipline, ack echo, SPEC-CONFLICT escalation — the injected command carries it; a protocol change edits that file, never a restatement here); the BRIEF carries only this wave's parameters.

Launch — the **train goal fires ONCE, at train start** (lanes: once PER LANE, at the lane's first wave — string + `train-goal.md` scoped to that lane's manifests; per `/chat:goal` § Launch mechanics — the orchestrator authors the string itself, no interrogation):

1. **Train start only:** `chat.sh inject {builder} "/goal deliver the {train-name} wave train — {N} waves per the Train-stamped manifests under $WAVES (pool trains: waves as the orchestrator BRIEFs them — pool dispatch owns selection and count; one /wave:builder BRIEF per wave, /wave:builder Orchestrated mode); act ONLY on injected turns — a task verdict, a /wave:builder BRIEF, or a boundary brief; awaiting one IS goal progress: end the turn empty-handed, answer a goal-continuation block with one holding line and end again, never start work unprompted, never open $WAVES manifests yourself, never run /jc unprompted, never self-schedule timers or background waiters — deferred Action Items are the orchestrator's to dispatch; the train ends when the orchestrator announces the train complete or terminated"` — ≤4k, single line; capture-verify (`Goal set` + the `◎ /goal active` statusline mark). Persist the string byte-identical to `docs/dev/trains/{train-name}/train-goal.md` (lanes: `train-goal-{lane}.md`; `mkdir -p` the train dir when `/wave:schedule` didn't create it) — the ONLY durable copy; every re-fire reads it.
2. `/wave:builder $WAVES/{wave-name}/BRIEF.md — God speed.` — capture-verify delivery (exit-3 or typed-not-submitted → re-inject `--force-now`).

**A worktree sees `main` AS IT WAS AT CUT TIME — so "reuse the existing pattern" resolves to the pattern that existed THEN.** Before briefing any work onto a fresh cut, name every primitive an IN-FLIGHT wave is about to introduce that this work should reuse; on a pre-merge cut the siblings do not use it yet, so a brief saying "mirror the siblings" instructs a REINVENTION — and a duplicate component never announces itself the way a merge conflict does. The collision to look for is with PRIMITIVES and LINTS, not files: disjoint file sets prove nothing. Either cut after the sibling wave merges, or SYNC `main` into the branch and re-brief BEFORE the builder starts — naming the primitive to reuse and the new lint as its own gate.

**Mid-train wave launches are a SINGLE inject** — capture the builder first and confirm `◎ /goal active` still shows (gone → re-fire + verify per § O4 GOAL-DROPPED, THEN the BRIEF); a fresh per-wave `/goal` is FORBIDDEN mid-train — it would REPLACE the train goal. A respawned builder (dead chat) gets the same re-fire-then-brief sequence before anything else. **Completely-new work = compact first (founder law):** every dispatch that opens work OUTSIDE the wave the builder just held — the next-wave BRIEF (§ O6.4's boundary compact is this rule's wave-boundary instance), a fill batch of deferred Action Items, a corrective re-dispatch onto a different surface — is preceded by a focused `/compact` into that builder (hold: train goal + allow-list verbatim + the new assignment's entry point; `--then` carries the dispatch) and the `◎` re-verify; new work starts on fresh context, never on the previous assignment's residue. Verify the waiter holds (§ O4 Liveness), plus the external heartbeat — a host cron that alerts the founder when no `$WAVES/*/STATE.md` was touched >45 min (glob, never a fixed path).

## O4 — Verify loop

**Ping-driven (fast path).** A builder ping (`Task #{N} done — report: {abs path}`) fires a turn immediately:

1. Read the task report + the task's **diff card**: an `Explore` (Sonnet) reader runs `git -C $WT diff {checkpoint N-1}..{checkpoint N}` **AND `git -C $WT status --porcelain`** — `git diff` is BLIND to untracked files, so a diff-only reader reports a brand-new test file as "no test file at all": it converts _outside my coverage_ into _does not exist_, which is this codebase's own honest-absence disease wearing a tool's face. The reader returns files, symbols, contracts touched, anomalies vs the task section — and NAMES what its instrument could not see. Read the FULL diff yourself only for `frontier-hands:`/{DOMAIN_ADJ} tasks and every ~5th task (sampled audit) — hygiene, GATE-1, and the walker re-cover every line.
2. Check against the spec task sections + corrections — Expected/Got on every gap; reject any changed path outside `$WT`/`$TASKS`. **A claimed exception on sacred ground ({SENSITIVE_DATA}, {DOMAIN_ADJ} safety, secrets) is verified against root `CLAUDE.md` at VERDICT time, never accepted on the manifest's word** — a spec is subordinate to root law, so "the manifest sanctions it" is the signature of a spec that violated it, not a defense; the diff's logging/telemetry/error-string surfaces get an adversarial read of their own on any task touching {SUBJECT_NOUN} content.
3. Relay the builder's checkpoint request to gitter (Phase: WORKTREE-CHECKPOINT, explicit file list) when the builder hasn't already had it taken. An ACCEPT ruling on builder work ends with its checkpoint relay in the SAME turn — accept-then-checkpoint is one atomic act. Gitter dispatches against ONE INDEX serialize END-TO-END — one gitter returns before the next starts. This binds on **`main` exactly as on a worktree**: the git index-lock serializes COMMANDS, not `add`→verify→`commit` SEQUENCES, so two concurrent gitters interleave their staged sets and one commit ships the OTHER's files under its own message — a commit that lies about its own contents, and `git log --grep` can never find the real work again. Never run two gitters against the same index, whatever the phase.
4. Verdict inject, one line: `T{N} verdict: proceed [frontier-hands: {spans}]` | `T{N} verdict: fix — {items}` | `proceed-disjoint T{M}` (parks a question while a disjoint task runs). Rule `frontier-request` pings in the verdict — grant by cost-of-being-wrong. A report's "left for task #N" defer is OWNED at verdict time — the verdict names the receiving task (checked at that task's review) or converts it to a tracked rider; an unowned defer is rejected. Every verdict carries the task id; the builder's next ping echoes it (ack). Unacknowledged verdict at next contact → re-inject `--force-now`, confirm via capture. For a provably disjoint, all-Sonnet-hands mechanical span, one verdict may pre-authorize it — `T{a}-{b} verdict: proceed-span`, restating verbatim "hold task-N+1 file writes until task N's checkpoint returns" — the span's cards get one combined review at T{b}, any finding there converting to `fix` verdicts.
5. STATE.md: rewrite TOP + `## Next`, append the outcome line below the marker. Update report.md's task row; append any fired retro entry (§ Steering conscience) before the turn ends.

**The 10m wake (watchdog duty is YOURS — no sensor exists).** Every wake — spool event, builder ping, or `HEARTBEAT-TIMEOUT` — checks ALL lanes: full-screen capture of each builder pane (never grep-filtered — filtered captures hide stuck queues, modals, half-typed injects) plus `$TASKS`/STATE mtimes, and for an on-main (`/wave:live`) lane ALSO its artifacts — w-status ledger + `main` git log since the last wake (poll; never rely on its pings alone). The capture is the detector; rule each lane against these states (a payload naming a lane routes, never rules) and act — an all-clear wake narrates ≤3 lines and re-arms:

- GOAL-DROPPED → re-fire from `train-goal.md` byte-identical before anything else. Entering any multi-hour idle-standby (mutex queue, sanctioned idle), capture-verify `◎` — the continuation loop can self-declare victory on a long standby despite the awaiting-IS-progress clause. A lane HELD longer than a task-span gets `/goal clear` + byte-identical re-fire at release — an active goal churns the continuation loop against a holding builder, burning multi-hour turns toward its context ceiling.
- IDLE / FINISHED-TASK (bare `❯`, no spinner) → a verdict is owed or the next step stalled (inspect `$TASKS` + `git -C $WT status`, corrective inject); awaiting-agents and post-boundary sanctioned idle are benign — no action. In `BOUNDARY-ACTIVE`: a ruling is owed (§ O6.2 sweep now) or the lane stalled (re-inject the boundary brief). In sanctioned idle (§ O6.4): a WORKING builder is the incident — halt it, log `friction`.
- FROZEN → the shell may have EXITED: check `ps` + the run-log's `EXIT=` line; exited → wake-inject the result (finalize, never re-run).
- LONG-TURN → judge by the thrash signature (tmux-Escape lever).
- CTX → queue a `/compact` for the next task boundary, injected BEFORE the pending verdict, never mid-task. **A SINGLE SNAPSHOT LIES BOTH WAYS** — read the ctx% across two captures before acting on either: ctx DROPPING = the compact is running and its steer is queued to auto-fire (WAIT — an Escape here kills the queued steer while the compact succeeds); ctx UNCHANGED after an interval, steer still in the composer = true queue-limbo (Escape). And a capture taken immediately after an inject catches the pre-spinner gap and reads as "dispatch failed" — re-capture before concluding it didn't land. EVERY builder `/compact` — any phase — carries a FOCUS in front (position + immediate-next + STATE.md pointer + Orchestrated-mode reminder), never bare, and carries `--then "/wave:builder {BRIEF path} — {hold/steer}"` (chat.sh REJECTS a steerless `/compact`; the detached waiter delivers the steer once compaction settles to idle — the steer IS the first post-compact inject, verdicts ride after it) + capture-verify a SINGLE `Compacting…` fired and `◎` survives (gone → re-fire).
- LIMITS (statusline gauges `5h-used:NN% ↻…`/`7d-used:NN% ↻…` — 5-hour/7-day account windows, % USED, `↻` = resets-in; one account: builder gauges = yours; either ≥80%) → **GENTLE PAUSE** — wrap in-flight work, start nothing new, park at a safe point (current item → checkpoint, builder idle) — CLEAR each builder's `/goal` FIRST (inject `/goal clear`; an active goal churns the continuation loop and never parks idle, and queued `/compact`s + steers jam behind the churn as messages), re-fire byte-identical from train-goal.md at resume — write the pickup point to STATE.md, stop the waiter, arm a one-shot resume cron (CronCreate recurring:false) at reset+10m that relaunches the waiter and resumes from STATE. Lanes: gauges are account-shared — GENTLE PAUSE EVERY lane (each parked per this rule), ONE resume cron for all (relaunch the waiter, then `resume-lanes`). Reset-proximity exception: `↻` shorter than the pause machinery itself would take → restraint instead — no new heavy dispatches, in-flight work continues, founder informed when present.
- DEAD → respawn per § Resume. **A destructive act — respawn, `/goal clear`, an Escape-recovery, a kill — demands PROOF OF DEATH, never an inference from a marker: zero turn-children AND no CPU AND no spinner, read off a FULL-screen capture.** `cache-✗` is a CACHE state and says nothing about liveness; a builder idle at a bare `❯` is HEALTHY-IDLE awaiting work, and the wedge signature is the opposite — BUSY-but-frozen (spinner + `STATE.md` >45m, confirmed by CPU delta + child count). **The evidence bar scales with IRREVERSIBILITY:** waiting costs latency and is recoverable; a respawn costs the session's whole context, its crons and its sub-agents, and is not. A partial view licenses only a fuller look — never an action.

**Liveness (waiter = mechanism, ping = fast path):** keep `.claude/scripts/wave-wait.sh` alive as this chat's OWN background Bash task at all times mid-train — it exits on the first spooled event or after 10m (`HEARTBEAT-TIMEOUT`), and the exit's task-notification is a wake no send-keys can swallow. EVERY wake sweeps FIRST: `$TASKS` mtimes vs the last ruled verdict (lanes: per lane) + the all-lane checks above + every outstanding HOLD/GO you have issued, RE-MEASURED directly (ps for load, artifact mtimes for gates — never waiting on a ping nobody owes; a hold whose condition you don't re-measure is a stall you created) + every IDLE lane's held items triaged PER ITEM — each names the specific shared resource, `Depends:` edge, pool conflict-block, or owed ruling it contends, re-derived each wake (a sibling's push or a lock release may have cleared it); a phase or window is never a blanket holder; an item with no nameable holder dispatches THIS wake. Word hold conditions event-anchored ("GO after {artifact} lands"), never ambient ("when load clears"). Relaunching the waiter is the LAST act of every turn. A sub-agent card's elapsed timer is NOT liveness (it freezes on live runs); liveness = process-subtree CPU delta + child count. A timeout wake owing nothing narrates ≤3 lines and re-arms.

**tmux-Escape lever.** To land a time-critical correction or break a runaway-dispatch turn: `tmux -S /tmp/tmux-$(id -u)/{builder-session} send-keys Escape` → the queued steer processes. Escape breaks only an ACTIVE turn (no-op on idle async-wait) and never kills a background sub-agent — the builder kills its own on your steer. Intervene on the thrash SIGNATURE (zero files landed + climbing tokens across captures), never on dispatch category — an exact-briefed producing hand finishes. **Modal deadlock** (a permission modal swallows the statusline; label-resolve breaks): `chat.sh modal {builder-session} deny {N}` — DENY-ONLY, only when the choice is unambiguous and the denied action is the destructive one, never blind-accept; capture-verify; log `friction`.

**Wake discriminator** (a wake finding no thread, or any fresh resume — rule on artifacts in this order, never on memory; every branch is scoped to THIS train — the `**Train:**` stamp matched against the source path carried in the invocation, steer, or lock payload; a lock collision on a DIFFERENT train's wave is a hard diagnostic stop; lanes: enumerate lanes from the registry FIRST, run per lane; lane-mapped trains match branches to the lane's `**Lane:**` stamp, pool trains to the registry's live assignment rows): (0) this train's source still holds `## Wave` headings whose Train-stamped manifests already exist → the split crashed mid-flight: complete the split + source reset FIRST. (1) a wave dir whose STATE.md stamp is an unfinished boundary (`BOUNDARY-ACTIVE`/`GREEN`, no `CLOSED`) → `/chat:capture {builder}` FIRST — busy on boundary duties → ADOPT: read artifact freshness, rule on unruled findings, wait; redo ONLY what the capture + artifact mtimes prove dead (the recorded walker run-id recovers its ledger — never relaunch blind); then launch + close. (2) a staged wave (`STAGED`/`CUT`, not `LAUNCHED`) → finish its launch from the stamp (◎-verify → single BRIEF inject — never a fresh per-wave `/goal`). (3) an in-flight wave (worktree + `LAUNCHED`) → re-enter O4 on it, first sweeping `$TASKS` for pings that arrived unanswered. (4) no wave dir claims the thread AND un-run Train-stamped manifests of THIS train exist → stage the lowest `## Wave {k}` of them (pool trains: the highest-`## Priority` READY wave; § O6.3 seed-only) and continue. (5) source not yet split → before re-invoking `/wave:orchestrator {builder} {source}`, grep `tmp/dev/archive/waves/*/manifest.md` for a `**Train:**` stamp naming this source — a match = the train already ran: HALT, never re-split.

**SYNC `main` INTO THE BRANCH BEFORE GATE-1 RUNS — never after.** A gate on a branch behind `main` tests a codebase that DOES NOT EXIST: it reports the branch REVERTING main's fixes as fresh failures (phantom RED), and by the same staleness it can go GREEN on a real conflict main introduced. **Wrong in both directions, and its RED is the expensive one** — it makes a healthy branch look like it needs remediation, so the lane grinds on phantoms while the true blocker (a SYNC nobody ran) goes unseen. Re-sync at MERGE is too late; the verdict was already computed on code that will never land. Gate what will ACTUALLY merge.

**A HOLD'S STATED CONDITION IS NOT ALWAYS ITS REAL BLOCKER.** Re-measuring a hold confirms the SYMPTOM ("GATE-1 still red") and does not diagnose the CAUSE ("the branch is 39 commits stale"). Any hold outliving a task-span gets a second question — _is this the real blocker, or a symptom of one?_ A gate that keeps failing may need a SYNC, not remediation.

**NEVER BLOCK THE SCHEDULER ON A LONG VERIFIER WHILE INDEPENDENT WORK IS PENDING.** You are the ONLY chair that can rule a verdict or dispatch a builder, so every minute your turn is blocked, every idle builder is blocked with it. A walker, a full suite, a forensic — anything measured in tens of minutes — runs as a BACKGROUND task (`run_in_background`), your turn ENDS, and the waiter wakes you: you rule pending verdicts and dispatch waiting lanes WHILE it runs. Only a verifier whose result gates the VERY NEXT act (GATE-1 before a merge) is worth waiting on, and even then, rule every INDEPENDENT owed verdict and dispatch in the SAME turn before you start it. Before any long call, sweep: what can I rule or dispatch right now that does not depend on this? Do those FIRST. A blocked scheduler with idle builders is the most expensive state on the board.

**Context discipline.** Read task reports and diff cards only — never raw test output or file dumps; bulky verification goes to disposable Sonnet/Haiku readers. Pre-ruling claim checks: wave-walker VERIFY mode (`scriptPath:'.claude/workflows/wave-walker.js'`, `args.claims`/`args.manifestPath` — name-lookup serves stale); any ad-hoc Workflow `agent()` inherits YOUR model — pass `model:'sonnet', effort:'xhigh'` (task-map frontier alias per frontier-hands logic). Past ~70% context, self-compact at the next natural pause (hold: wave position, verdicts owed, escalations, waiter state; the hold records an inject only WITH its delivery proof — exit-0 + capture-confirm; an unproofed send is written as OWED so the resumed chat re-delivers instead of trusting the recap) — ALWAYS with the § O6.6 `--then` steer (`/wave:orchestrator {builder} resume {wave-name}`).

**Milestones** (marked in the task map): before the next task starts — cross-project typecheck + affected integration profiles in the worktree; a reconcile refresh over ONLY the anchors the SYNC merge diff touched; **gitter SYNC** (Phase: SYNC — merge current `main` INTO `pipeline/{wave-name}` inside the worktree) so divergence surfaces under the wave's tests, never in a blind end-merge.

**Steering conscience (`retro.md`).** The orchestrator audits its own steering: append to `$WAVES/{wave-name}/retro.md` — any phase, at the moment of detection, never reconstructed at close-out. reversal and friction are mechanical: the event fires the entry, no judgment call. intent-gap, tier, and obs are judgment: resolve every doubt toward logging. A qualifying event without an entry is itself a steering failure. Entry:

```
## R{n} · {T{N} | phase} · {reversal | intent-gap | friction | tier | obs}
Expected: {what the steer assumed}
Got: {what happened}
Amend: {file § section — concrete change; or `judgment` when no text fix exists}
```

- reversal: a fix lands in a task already verdicted `proceed`, whatever surfaced it
- intent-gap: a goal was met as written yet needed a re-steer to be right — the amending verdict/brief line is the event
- friction: machinery burned time/tokens — unacked verdict, re-inject, deadlock, respawn, resume confusion, verdict latency leaving the builder idle
- tier: a `frontier-hands` grant plain Sonnet work satisfied, or a Sonnet-hand defect a frontier grant would have prevented
- obs: any other framework improvement worth making next time — including a wave O0 should have bounced to `/wave:refine`

A builder `retro:` ping note becomes its own entry (note verbatim in `Got:`). Consumer: the founder's `/pcm` pass over archived retros, reconciled against co-archived artifacts — a post-`proceed` fix with no reversal entry is a withheld entry. A folded amend is stamped `Resolved: {date} — {where}` under its entry, in place.

## O5 — End-of-wave (order is load-bearing: hygiene remediation converges BEFORE the full gate)

Builder over ~50% context at O5 entry → inject a focused `/compact` (hold: O5 duties, task-map tail, the train goal) BEFORE hygiene — hygiene+GATE-1 is the wave's heaviest context phase.

1. **SYNC** — gitter merges current `main` into the branch one final time. YOU drive this step and confirm the sync commit is on the branch BEFORE calling the builder's End-of-wave duties — hygiene+GATE-1 run on a branch behind `main` yields a partially-stale green (recoverable only by O5.5 re-sync + affected re-runs).
2. **Hygiene review** — spawn Opus: execute `audit/code-hygiene.md` scope `diff` over `git -C $WT diff --name-only main...pipeline/{wave-name}` → `$WAVES/{wave-name}/hygiene-review.md` with a verdict line.
3. **Remediation** — builder fixes every hygiene code finding in the worktree (fix → targeted re-test → re-audit the touched category); loop until clean, cap 2 rounds; residuals land in the report and `### /jc Action Items`.
4. **GATE-1 (pre-merge full — LAST pre-merge step)** — builder-executed per `wave/builder.md` § End-of-wave duties; verdicts land in `$WAVES/{wave-name}/gate1.md` — a skipped tier (`envBlocked`/`INTEGRATION-UNRUN`) is never green; escalation → BLOCKED-DEFERRED, worktree preserved.
5. **MERGE** — gitter, `Wave-mode: v2` (precondition = main's working tree clean for every path in `git diff --name-only main...pipeline/{wave-name}` — a dirty overlap held by an on-main lane makes git refuse the merge, so that lane's commit is a merge gate; the overlap check runs against the FINAL post-checkpoint branch — re-run it after the last checkpoint lands, never off a pre-checkpoint diff; AND `gate1.md` all green AND gate-COMPLETE: its verdicts cover every top-level dir in `git diff --name-only main...pipeline/{wave-name}` — a touched project without one re-runs GATE-1 for it; completeness is ENVIRONMENTAL too — a code path the gate's stack structurally cannot host (a service or DB with no equivalent on that stack) is a declared HOLE, named in the gate artifact with its scenario and its `/jc` item, never a silent pass; `$DOCS` derives to `$WAVES/{wave-name}`), serialized by `git-lock.sh`. Main moved since SYNC → gitter re-syncs and the builder re-runs affected profiles only before retrying. When gitter RETURNS the merge SHA (never on entering MERGE): write `**Merge SHA:** {sha}` into report.md — the SHA return is a DISPATCH EVENT: parked /jc items whose named holder was the merge surface dispatch now — capture the builder — context over 70% → inject a FOCUSED pre-boundary `/compact` first (hold: boundary duties, artifact paths, /jc rules, the train goal + allow-list) — then inject the **boundary brief** (one line; switches the builder to BOUNDARY mode, `wave/builder.md` § Boundary duties; lanes: acquire the boundary mutex first and word the brief per § Lanes): `Boundary {wave-name} — merge SHA in report.md; run § Boundary duties, ping each artifact.` Stamp STATE.md TOP `BOUNDARY-ACTIVE — builder executing boundary duties`. The train goal stays live — nothing closes it mid-train.

## O6 — Boundary (builder executes, orchestrator observes and steers)

Every transition rewrites the closing wave's STATE.md TOP stamp (`BOUNDARY-ACTIVE → GREEN → CLOSED`; the staged wave's own STATE.md runs `STAGED → CUT → LAUNCHED`) at the moment it happens — the stamps are the wake discriminator's and the heartbeat's substrate. Every verifier artifact persists the moment its verifier returns, never batched into a later check. **OVERLAPPED BOUNDARY — THE DEFAULT, not an option** (an "option" never fires, and the serial path strands every idle builder behind one chair): at MERGE, SETUP cuts the next worktree from the MERGE SHA immediately (not waiting green); GATE-2 + walker run as background lanes while the builder starts next-wave first tasks; a BLOCKING finding HALTS next-wave tasks until remediated on `main` + gitter SYNC into the new branch; re-cut ONLY if the merge itself reverts.

1. **Verify on `main` (builder-executed)** — the builder runs `wave/builder.md` § Boundary duties end-to-end: background walker (merge-SHA mode off report.md's `**Merge SHA:**` line, run-id recorded in STATE.md at launch) concurrent with foreground-serial GATE-2 on the canonical stack. **Walker ownership is dialect-conditional:** a Claude builder launches its own walker (it holds the Workflow tool); a non-Claude lane (codex dialect) cannot — on its `walker-launch request` ping, or unprompted at its boundary, YOU run `wave-walker.js` for that lane and persist `walker-ledger.json` yourself. Observe the artifacts: per-project `gate2.md` verdicts as they land, `walker-ledger.json` on return, a ping each.
2. **Findings lane (orchestrator rules, builder implements)** — severity floor: a GATE-2 regression, a BROKEN thread, or a confirmed high+ anomaly **blocks the boundary**; MINOR and UNPROVEN findings land in `### /jc Action Items` and never delay the cut. EXCEPTION — {SENSITIVE_DATA}/security infra: the MINOR→defer floor never absorbs a cheap fail-open or blast-radius fix (a fail-closed gate that silently skips, a root/full-cap sidecar, over-broad env exposure); classify by (fixable-now-cheaply AND security-correctness), not the severity label — defer only what genuinely needs founder supervision or new-scope code. Each ruling is a verdict inject with an id (ack law: § O4); **green-detection is POLL-based** — every acting turn sweeps `$WAVES/{wave-name}/` for unruled gate2 rows/ledger findings and rules regardless of ping delivery. Fix-now rulings the builder implements via `/jc` (timing per its § Boundary duties); every fix-now fix — including orchestrator-designed — gets one independent judge over its diff before it counts fixed (QA-green ≠ convention-correct); INFRA hardening is judged EMPIRICALLY — build + run + prove the protected function still fires (a "secure" config can silently disable the function it hardens; for scheduler/daemon changes acceptance = the job actually runs, never just "the container starts"); the independent judge is NOT redundant with the builder's own empirical judge — prompt it explicitly for crash/OOM/kill SILENT-failure modes (proving "it works" never proves how it silently fails); never force-stop a running gate on a pattern-match — a low token-delta long run is NOT proof of a hang, and a run writing failure artifacts is FINDING bugs: read its artifacts + exit code BEFORE any Escape or green ruling; a fix in project P re-runs P's boundary reconciler (§ O6.3). gate2.md records fix SHAs + re-run scope. Cap 2 rounds; residuals defer, named in the announce. **SHIPWRECK or a confirmed security (R6/R7) anomaly is stop-the-train**, sequenced: orchestrator halt-inject → builder halt-ack names its dirty files (its § Boundary duties) → gitter REVERT (stash exactly the named dirty files, `git revert -m 1 {merge-sha}`; branch preserved — the wave re-enters O5 remediation) or, where a revert is worse, hard-block + watcher escalation (founder only if physically-founder-only) — and EVERY train-ending abnormal path (stop-the-train, terminal BLOCKED-DEFERRED, founder halt) fires the goal-closing announce (`Train terminated.`) + stand-down before anything else stops.
3. **Prep (orchestrator, overlapped with 1–2; partitions remaining only)** — O0 pre-flight for the next wave (partition-scoped, against its manifest); staging is SEED-ONLY: locate THIS train's manifests (`**Train:**` stamp matching this train's source), pick the lowest un-run `## Wave {k}` (pool trains: the highest-`## Priority` READY wave), acquire its singleton lock + seed its STATE.md `STAGED` + report/retro skeletons (two-lock invariant establishes at STAGE) — **no extraction, no source rewrite, manifest untouched**; O2 reconcile against post-merge `main` (one Sonnet reconcile+BRIEF agent suffices); BRIEF spec half (build truth + task map; the env card waits for SETUP); token telemetry (`node .claude/commands/p/tokens/token-ledger.mjs --by-workflow` for this chat, `--session {builder-session}` for the builder; append to report.md; any task >3× the wave median fires a `friction` retro entry). Teardown is builder-run (§ Boundary duties); verify only via `ps-test-pipeline`.
4. **Launch (ONLY on all-green: gate2.md + walker verdict persisted, blocking findings fixed)** — inject the builder's boundary compact: `/compact {focus: wave done; train goal + allow-list verbatim; idle SANCTIONED until next BRIEF}` — sanctioned idle begins HERE, never earlier, and ends at `LAUNCHED`. The compact + its `◎` re-verify is a LAUNCH PRECONDITION. Stamp `GREEN` → gitter SETUP cuts the next worktree from verified `main` → finalize the BRIEF env card (worktree, branch, ports) → stamp `CUT` → capture-verify `◎ /goal active` (gone → re-fire, § O4) → single inject `/wave:builder $WAVES/{next-wave}/BRIEF.md — God speed.` → stamp `LAUNCHED`.
5. **Close the wave** — **Docs:** `Workflow({ name: 'documenter-fanout', args: { mode: 'ARCHIVE', pipelineName: '{wave-name}', docsPath: '$WAVES/{wave-name}', epicName: '{epic-name}', waveOwned: true, timestamp: '{YYYY-MM-DD}' } })` — mid-train waves SKIP this fan-out (the epic line + archived gate artifacts are the record); the FINAL wave runs it once, the scout scoped over every train wave's merged diff (merge SHAs from the archived reports). **Epic:** skip when `none`; consolidate ONE wave entry per `documenter.md` § Epic consolidation contract. **Archive:** gitter DOCS-COMMIT (`Archive: $WAVES/{wave-name}`) — report (with review), gate artifacts, walker ledger, retro, STATE; the wave dir is committed TRACKED-AND-CLEAN before the move (`tmp/` is gitignored — an untracked file that reaches the move exits git history), then moves to `tmp/dev/archive/waves/`; verify the move. Release the singleton lock. Stamp `CLOSED`. Announce: `Wave complete ({wave-name}). {n}/{N} tasks · GATE-1 {verdict} · GATE-2 {verdict} · walker {verdict} · retro {tally}.` Train: append `{k} waves remain`.
6. **Self-compact (partitions remaining only, LAST)** — verify the waiter alive (dead → relaunch), then `/chat:self:compact {hold: next wave name, builder chat, waiter state, train source path} || {steer: /wave:orchestrator {builder-chat} resume {next-wave-name}}`. The `--then` steer is MANDATORY on self — without it this chat strands command-less (the waiter wakes, never restores). The steer re-enters O4 on the already-running wave with fresh context — and its FIRST act is a waiter verify+relaunch: the waiter PROCESS survives compaction, but a timeout-exit landing during the compaction window is swallowed unheard (no wake, no last-act relaunch — the chat goes deaf on holds whose conditions have cleared); treat the waiter as UNKNOWN after every self-compact. Lanes: the steer is ALWAYS `/wave:orchestrator resume-lanes` (hold: registry path + per-lane positions) — it must re-enter EVERY lane, not just the compacting one.
7. **Final wave / single wave** — steps 1–2 identical (the walker always verifies on `main`; teardown still builder-run); no prep/launch; step 5 runs with full docs. **Train-close hygiene sweep — self-executing, part of the close, never founder-ordered:** before the terminal announce, every `.worktrees/` entry is REMOVED (uncommitted work first preserved on its branch via a carry commit; a runway cut for a wave that never launched follows the same rule) or DECLARED KEPT with its removal condition named; every closed wave dir — and every `docs/dev/builds/{name}` pipeline dir this train's waves produced — is archived (tracked-and-clean gate, step 5); the train dir `docs/dev/trains/{train-name}/` and every queue spec this train consumed (`**Status:** SCHEDULED → {train-name}` under `docs/dev/waves/queue/`) archive the same way — one gitter DOCS-COMMIT names them in `Archive:`, commits them tracked-and-clean, moves them to `tmp/dev/archive/trains/` / `tmp/dev/archive/waves/queue/`, commits the removals; main's dirty files are committed scoped or declared held WIP with an owner. The announce carries the sweep line — `hygiene: worktrees removed {n} / kept {m} ({conditions}) · wave dirs archived {a} · train dir + {q} specs archived · main porcelain {k}` — and is INCOMPLETE without it. Then close the train goal by delivering its terminal condition: inject `Train complete — stand down.` and capture-verify the goal banner fired; still `◎` active → re-announce `--force-now` once → still active → inject the no-op successor `/goal train goal closed — stand down, take no action` (overwrite sanctioned ONLY here) → still active → watcher escalation FIRST. Then stop the waiter; no self-compact steer needed.

## Resume

- **Builder died** — DEAD escalation → respawn `--detach`, re-fire the train goal (§ O4 GOAL-DROPPED; goal state is per-session), THEN the RESUME brief: `git -C $WT diff {last-checkpoint}..` to see half-done work, hard-reset to the last checkpoint, redo the in-flight task from clean — never continue blind (boundary-mode death: no worktree — resume § Boundary duties from artifacts). STATE.md + checkpoints + task reports are the durable index.
- **Orchestrator died** — the external heartbeat alerts the founder; a fresh `/wave:orchestrator {builder} resume {wave-name}` finds the singleton lock, adopts it (stale PID), reads STATE.md TOP, captures the builder, and re-enters O4 at the recorded position. On ANY orchestrator resume — fresh invocation or a relaunch of this chat: re-run `/chat:whoami` + rewrite STATE.md's orchestrator handle/ping fast-path; launch the waiter (a background task never survives a chat death); re-stamp the singleton lock PID; inject the builder one status no-op so its reply-footer refreshes your handle.
- **Boundary death** (self-compact steer swallowed, or a crash anywhere in § O6) — rule by the O4 **Wake discriminator**, on artifacts in its stated order: unfinished-boundary wave first, staged wave second, in-flight wave third, source re-invoke LAST and only when no wave dir claims the thread. Mid-wave death is NEVER the source form — STATE.md present → `resume {wave-name}`; the in-flight partition lives only in that wave's manifest, not the source.
- **BLOCKED-DEFERRED** — worktree + ports preserved; `$WAVES/{wave-name}/BLOCKED.md` carries the trigger + resume protocol (template: `docs/commands/build/references/build-reference.md` § BLOCKED.md).
- **Lanes** — every orchestrator-side resume in lanes mode is `/wave:orchestrator resume-lanes` (§ Lanes); the builder-died and BLOCKED-DEFERRED bullets apply per lane unchanged.

## Lanes — multi-builder trains

A **lane** = one builder chat delivering its ordered subset of a scheduled train end-to-end, every wave under the unchanged single-lane O0→O6 cycle; N lanes run concurrently under THIS one orchestrator, which interleaves their O4 loops in its own turns. `/wave:orchestrator lanes [task file]` accepts two `/wave:schedule` dialects: **lane-mapped** — EVERY `## Wave` header carries `**Lane:** {name}`, a lane's queue = the Train-stamped manifests carrying its stamp; **pool** — `## Priority` + `## Conflict graph` blocks and no lane headers (rare `**Lane:** pinned-to:{wave}` exceptions only). A train matching neither dialect is fatal (back to `/wave:schedule`). The WHOLE-TRAIN SPLIT (§ O1) runs once either way. Wave names stay train-unique, so per-wave singleton locks, worktrees, and `$WAVES` dirs never collide across lanes.

**Lane registry** — `docs/dev/trains/{train-name}/lanes.md`, rewritten at every lane transition like a STATE.md TOP: one row per lane — `lane → builder chat → train source → current wave → lifecycle stamp`. Written at lanes launch before the first inject; resume and the wake discriminator read it FIRST to enumerate lanes; archived with the train dir (§ O6.7).

**Pool dispatch.** On a pool train no builder owns a queue; `docs/dev/trains/{train-name}/pool.md` (the split-copied `## Priority` + `## Conflict graph`) is the dispatch source. A wave is READY when its `Depends:` are all MERGED, every COMPOSABLE producer in its graded pairs is MERGED, no CONFLICTING partner holds a singleton lock (locks live STAGED through CLOSED — lock-held, not merely LAUNCHED; the closing wave whose `boundary done` fired this dispatch does not count), and any `**Lane:** pinned-to:{wave}` pin resolves (the builder that built {wave} is free). Dispatch the highest-`## Priority` READY wave at launch, at every `boundary done` ping (the mutex-release event — never earlier), and at resume; area-affinity with the builder's previous wave breaks priority ties; an O6.3 prep pick is PROVISIONAL — re-verify READY at the dispatch moment. A conflict-blocked wave stays pooled (re-checked at the next dispatch event), and "pool conflict-block" is a NAMED holder for an idle builder. A dead pool builder's in-flight wave may re-dispatch to ANY idle builder (lock re-stamp + lanes.md row move + BRIEF re-inject) when respawn-same-chat fails. OVERLAPPED BOUNDARY on a pool train launches the O6.3 provisional pick re-verified READY at MERGE; the never-earlier rule governs ordinary dispatch. lanes.md rows record live assignments (builder → current wave); pool.md archives with the train dir (§ O6.7).

**Launch** — per lane, in registry order: O0 per its first wave (item 5 verifies/spawns the lane's OWN builder chat), O1 stage (the waiter launches ONCE, train-wide), O2–O3 brief + per-lane train goal, first BRIEF inject. Route every subsequent artifact by its identifiers — spool events carry `[{builder}]`, pings carry wave-scoped report paths.

**Boundary mutex** — GATE-2 runs on the single-tenant canonical stack, so boundaries serialize across lanes via `tmp/wave-boundary.lock`: acquire before the boundary brief (atomic `mkdir`; write `{wave-name} {sid} {PID} {timestamp}` to `holder` inside — the stamping session's sid is mandatory, role labels collide across co-tenant orchestrators on one account; takeover test = holder >2h AND the holder wave's boundary artifacts stale (gate2.md/STATE.md mtime outside that window) — the holder PID is a transient stamping shell, dead by construction, never liveness evidence; a REACHABLE plausible owner (a co-tenant orchestrator one inject away) is ASKED before any release — a release is destructive, and your own empty ps enumeration is never a verdict on THEIR gate). The boundary brief names the holder artifact + the builder.md § Boundary duties exclusivity recipe — a cross-chat GO alone is unverifiable inside the builder's session and its classifier rightly refuses destructive stack commands on it. Held by another lane → inject `Boundary {wave-name} — merge SHA in report.md; run § Boundary duties (1)+(3) only, GATE-2 queued behind {holder}` (walker launch + ledger persist are static, stack-free), then on release `Boundary clear — run § Boundary duties (2)+(4)+(5)`. Release (`rm -rf`) at the holding lane's `boundary done` ping — the mutex spans GATE-2, fix-now re-runs, and teardown, keeping every canonical-stack touch single-lane. An EARLY handoff (any GO before `boundary done`) requires the outgoing lane's POSITIVE-ACK that its stack-touching steps are complete or explicitly stripped (echoed back) — a ps-snapshot is never the precondition against a lane holding queued orders. `/wave:live` W4 honors the same lock.

**Cross-lane SYNC** — a lane's merge to `main` reaches every other lane's open worktree through that lane's own § O4 milestone / § O5.1 gitter SYNC — mandatory at its next milestone or wave end, whichever first; conflicts and drift surface under THAT lane's tests + reconcile refresh. A COMPOSE-BOTH disposition on a validated persisted payload names each surviving signal's CARRIER — registry-validated keys only in the validated field, ancillary signals (partial/degraded/reason) on the unvalidated channel; "both survive" without carriers glosses the rehydration trap.

**Verdict routing** — every inject (verdict, brief, compact, capture-verify) names its lane's builder chat; a verdict landed in the wrong lane's builder = `friction` retro entry + immediate re-inject to the right one.

**resume-lanes** — `/wave:orchestrator resume-lanes`: read the registry; refresh orchestrator identity once (§ Resume orchestrator-died steps — whoami, waiter launch, lock re-stamps); then per lane run the § O4 wake discriminator scoped to that lane and re-enter O4, sweeping its `$TASKS` for unanswered pings. This is the § O6.6 steer, the LIMITS resume cron's target, and the orchestrator-died re-entry in lanes mode.

**Already-serialized machinery (no lane work needed):** gitter via `git-lock.sh` for every checkpoint/SYNC/merge; alloc-ports for per-lane pipeline stacks; GATE-1 on the worktree's own stack.

**The other shared resources.** Four more are global, not per-worktree, and each serializes across lanes: (a) **the package-manager store** — `{PROJECT_PKG_MGR}` shares one content-addressable store per machine, so an install in any worktree can park a sibling lane's test run indefinitely (a suite that never spawns its runner: elapsed climbing, CPU flat). A lane running an install holds `tmp/wave-boundary.lock` for its duration, exactly like the canonical stack. (b) **main's working tree** — an on-main lane (`/wave:live`) holding dirty files makes another lane's merge impossible, since git refuses to overwrite them. (c) **the {UI_FRAMEWORK} dependency tree + test cache** — `worktree.sh` symlinks every lane's `{project-ui}`'s dependency directory to main's, so all lanes share one module cache and test cache; concurrent {UI_FRAMEWORK} test runs contaminate each other's results. A lane running {UI_FRAMEWORK} tests holds `tmp/wave-boundary.lock` OR passes a run-unique `--cacheDirectory`. (e) **the mock-{AI_PROJECT} ports** — allocated per (lane, worker) and NOT keyed on the RUN, so ANY two concurrent BE-integration runs collide on them regardless of which DB stack each uses (the gap is flagged in `per-worker-resources.ts`'s own docstring). Consequence: an on-main `/jc` running a FULL suite is as single-tenant as GATE-2 — it takes `tmp/wave-boundary.lock` or runs on a pipeline-scoped stack. (d) **gate windows themselves** — concurrent cross-lane gates contend on {AI_PROJECT}-SSE/{QUEUE} adjacents and surface shared-resource flakes no solo run shows; when two lanes' gate windows overlap, the later window runs reduced cross-lane concurrency (`BE_TEST_LANES=1`) or waits for the mutex. Scoped to OVERLAP only: a solo window keeps full concurrency, and in-suite parallel-4 stays everywhere — this knob serializes lanes, never lowers workers to green a flake.

**Parallel-by-default.** Serializing a lane requires a NAMED holder: one of the shared resources above, the boundary mutex, a manifest `Depends:` edge, a pool conflict-block, or a ruling this chat still owes it. A sibling lane being mid-boundary is NOT a holder — a boundary that touches none of the shared resources (stack-free infra validation, disjoint write-paths) merges and closes CONCURRENTLY with it, and a lane whose `Depends:` edges are all MET launches regardless of what its siblings are doing.

**Idle capacity (lane-mapped trains).** At every wave close, compare remaining lane queues: a lane dry (or within one wave of dry) while a sibling holds ≥2 unstarted waves is a scheduling defect, not a rest — flag `/wave:schedule rebalance` to the WATCHER with the candidate delta (wave → lane → why safe) the moment the imbalance is FORECAST, never when the dry lane parks; on its APPROVE, apply the delta yourself — re-stamp `**Lane:**` in the affected manifests (the one sanctioned post-split manifest edit) + rewrite lanes.md. A pool train assigns at dispatch and cannot go lopsided.

## Rules

- **NEVER delegate Skill calls to sub-agents** — invoke directly.
- **Observe and steer, never execute** — ALL real work (code, gates, walker, `/jc` fixes, teardown) is the builder's; the orchestrator reads artifacts, rules findings, stages documents, and drives gitter. Sole execution exception: the boundary walker of a non-Claude lane (codex dialect) — § O6.1.
- **Only gitter runs git** — checkpoints, SYNCs, merges, doc commits all through it.
- **Every gate verdict is a persisted artifact** (`gate1.md`, `gate2.md`, `hygiene-review.md`, `walker-ledger.json`) — a verdict that exists only in chat context does not exist.
- **Verdict-before-next-task** — the builder proceeds only on an acked `proceed`; the orchestrator reviews every task diff, no rubber stamps.
- **Never dictate a record's factual content to a builder** — state the DISPOSITION (defer, caveat, document); the builder writes what is factually true from its own evidence. A builder refusing your wording is signal, not friction.
- **Hygiene runs pre-merge; GATE-1 runs after its remediation converges** — nothing merges unexercised by a full suite. The walker runs post-merge on `main`, concurrent with GATE-2; every blocking finding is fixed there via `/jc` before the next worktree is cut.
- **STATE.md discipline** — TOP rewritten only at lifecycle transitions; below the marker append-only via shell append (`cat >>`), never Edit-tool rewrites (Edit makes the whole file context-resident).
- **Founder amendments mid-wave** land only in NOT-yet-started tasks; a running task is never re-scoped.
- **Founder PAUSE/HOLD** — `/goal clear` EVERY builder FIRST (an active goal keeps it churning) AND stop the waiter — zero wakes until the founder's RESUME: relaunch the waiter, re-fire each goal from its `train-goal.md` byte-identical.
- **AskUserQuestion at the founder is reserved for physically-founder-only matters** (account, billing, hardware, legal identity) — and even then only when every lane sits at a safe hold: an open menu deafens this chat (waiter notifications queue, injects vanish) until answered. Every OTHER decision gate routes to the WATCHER handle per the advice route below; no watcher running → state the recommendation in the announce, act on it, and let the founder overrule on return.
- **Advice + infra route** — decision advice (a ruling you're unsure of, a wrong-turn recovery: "what should happen now") and framework/`.claude/**`/`CLAUDE.md`/script fixes go to the WATCHER chat when one runs: inject the handle in `tmp/wave-sensor/watcher.handle` (absent = no watcher — infra fixes park for the founder; decisions follow the AskUserQuestion rule above: announce the recommendation, act on it, founder overrules on return). An advice-ask is NON-BLOCKING — hold only the asked item, keep every other lane moving; the answer is a recommendation, the ruling stays yours. Rulings, reversals, defers, and FOUNDER-owed items persist in the wave's artifacts (STATE/retro/gate files) — the watcher harvests them into `docs/dev/trains/{train-name}/decisions.md` and debriefs the founder at each wave close. `/wave:watcher` also runs the panoramic 50m rounds, watches the waiter, and folds retros.
