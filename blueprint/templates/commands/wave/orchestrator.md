# Wave Orchestrator — Dual-Chat Wave Runner

$ARGUMENTS

This chat is the **wave_orchestrator** — stages, briefs, verifies every task, gates the merge, closes out; the **builder** (first argument = its tmux name from `/chat:ls`) implements. Judgment never delegates down; implementation never re-decides the spec.

**Autonomous execution contract:** pre-flight (O0) is the only gate; once it passes, the wave runs to completion. Founder present → relay product-level SPEC-CONFLICTs; founder unreachable (God speed) → rule by best judgment on the reconciled spec, or mark that task BLOCKED-DEFERRED and continue disjoint tasks.

**Models:** founder-defined per chat; default **Opus orchestrator · Sonnet (xhigh effort) builder** — verify the builder pane's statusline model at every launch/respawn, correct via `/model` inject; a frontier-hands grant may flip it up per span, back after. Launch THIS chat with `ENABLE_PROMPT_CACHING_1H=1` — 1h cache, orchestrator-ONLY (write-heavy chats pay 2× writes); the sensor self-selects its tick by it.

## Resolve the task file

- **Empty/blank second arg:** task file is `wave.md` at repo root.
- **File path:** read that file. **Description:** parse as inline tasks (must still meet the ZERO-GAP bar).
- Extract `**Epic:** {name}` (`none` if absent; train — a partition's own `**Epic:**` header line overrides the top-level for that wave). Choose a kebab-case `{wave-name}` (2-4 words); verify uniqueness against `$WAVES/`, `docs/dev/builds/`, `tmp/dev/archive/{builds,waves}/` — collision → `-v2` or a more specific name.
- **Wave train:** tasks spanning more than one code impact area never run as one wave — the spec's `## Wave {k}:` headings + their header blocks (**Changes / Touches / Depends / Tasks** — `/wave:refine` § wave.md format) declare the partitions: build the train map by grepping the headings + the four `**Field:**` lines beneath each and split on heading line numbers mechanically, NEVER by reading the full source. Undeclared multi-area file → derive the partition map ONCE — a Sonnet reader returns per-task area + file-plan cards AND surfaces every preamble/rules block with the task range it binds — then write the `## Wave` headings + header blocks into the source before extracting, so every later wave and resume reads the same ruling; mid-train you may subdivide a partition that proves too big, never merge or reshuffle across declared boundaries. Founder override: remaining partitions may merge into ONE wave cycle — one worktree, one BRIEF spanning their still-READ-ONLY manifests, one ceremony. Each partition is a full wave (own worktree, own O0→O6 cycle, merged to `main`) before the next starts. This invocation runs the FIRST remaining partition; § O6 chains the rest. Preamble rides by scope: the `**Epic:**` header + sections binding ALL tasks go into every partition's manifest; a section binding a task subset (a later `# Tasks` block's preamble, a per-wave rules block) goes into exactly the partitions holding tasks it governs.

## O0 — Pre-flight

FAIL FAST before any setup. All fatals print a diagnostic and exit — no wave dir, no worktree.

1. **Singleton lock** — `mkdir tmp/wave-{wave-name}.orchestrator` (atomic) and write `$$ + timestamp` inside. Exists already → another orchestrator owns this wave: refuse, point at it. Released at the § O6 close (per-wave-name; a boundary holds two — closing + staged).
2. **ZERO-GAP shape** — every task carries the 10 refine sections (routing, data model, contracts, file plan, behaviors, boundaries, anchors, flow, why, summary) plus `**Build agents:**`. Loose prose → route to `/wave:refine`. **Train:** every `## Wave` heading must grep-recover all four header fields — a missing field is fatal (declared spec → back to `/wave:refine`; fallback → you wrote them, fix them).
3. **Existence + conflict checks** — grep every named entity (components, tables, endpoints, files); nonexistent referent or incompatible edits to one target across tasks → fatal. **Train:** scope items 2–3 to the current partition + preamble — an entity a prior partition creates resolves against merged `main` at that wave's pre-flight; cross-partition sequenced edits are ordering, not conflicts.
4. **Command triage** — `[CMD: /jc]` tasks run via `Skill("jc", ...)` now; `[CMD: /km]` via `Skill("km", ...)` BEFORE launch (builds may depend on the knowledge files). Any task whose file plan touches `{AI_PROJECT}/knowledge/` or `.claude/**` is split out here to /km//pcm — the builder never writes those paths, even when a gate marker is open. /km on WORKTREE knowledge files stamps the gate marker at the worktree's own `tmp/` (`$WT/tmp/professor_km_active` — km-guard anchors to the edited file's repo root; a worktree IS its own root); deletions follow km.md's checklist (file + REGISTRY.json entry + verify remaining paths resolve).
5. **Builder chat** — first argument names it; verify it exists and idles (`/chat:ls`). Missing → spawn `/chat:new --detach {wave-name}_builder` (own tmux socket = crash isolation). Record its name + your own (`/chat:whoami`) for the brief.
6. **Dirty main** — allowed; default `[CarryWIP: leave]`; gitter SETUP owns the mechanics.

## O1 — Stage

1. `mkdir -p $WAVES/{wave-name}`; copy the spec to `manifest.md`; **if the source is root `wave.md`, reset it to the `# Tasks` stub in the same atomic step**. **Train — the WHOLE-TRAIN SPLIT (first invocation only):** (a) gitter DOCS-COMMIT the pre-split source; (b) split EVERY partition into its own `$WAVES/{kebab-area}/manifest.md` — the `## Wave {k}:` heading + four header fields + a `**Train:** {source-path}` stamp line + its scoped preamble/rules blocks + its tasks, all line-range ops, never a full-file read; a manifest that already exists for a heading is SKIPPED, never rewritten; (c) verify the task-count sum across manifests against the source; (d) ONLY then reset the source to the stub. Manifests are READ-ONLY after the split — any later step that would rewrite one is a fatal diagnostic. Only the CURRENT wave gets item 2's STATE.md; later manifests stay STATE-less (inert to the wake discriminator and heartbeat) until § O6.3 stages them.
2. Seed `STATE.md` delta-structured: TOP = live resume brief + lifecycle stamp (`STAGED → CUT → LAUNCHED → BOUNDARY-ACTIVE → GREEN → CLOSED`, rewritten at each transition) + `## Next`; marker line `<!-- APPEND-ONLY BELOW — never rewrite -->`; below = locked decisions from refinement, then appended facts. Seed `report.md` skeleton (task table, gate slots, a `**Merge SHA:**` slot the walker's scout greps) and `retro.md` (header line only — the steering-conscience ledger, § O4). **Train:** after item 1's split the QUEUE is the set of Train-stamped manifests — a wave dir with STATE.md is current or done, one without is un-run; the source stub never holds partitions again.
3. **gitter SETUP** — `Pipeline: {wave-name}. Phase: SETUP. CarryWIP: leave.` One worktree `$WT`, branch `pipeline/{wave-name}`, ports + `.env.ports`, `ports.md`. SETUP also provisions the recurring gate-killers: copy gitignored `.env.local`s into the worktree, clear the {UI_FRAMEWORK} test cache, align `.env.test` ports to the allocated pipeline ports. **Train:** SETUP here is the first wave only — a boundary-staged wave defers it to § O6.4, so its worktree is cut only from verified `main`.
4. **Task map** — order the spec's tasks per its sequencing directive (a group = consecutive tasks). Mark **milestones**: the spec's declared milestone markers, else every group boundary, else every 4th task. Mark **`frontier-hands:`** on any span whose cost-of-being-wrong warrants a frontier-tier hand (novel algorithm, auth/consent path, concurrency, {DOMAIN_ADJ}-adjacent logic) — hand-tier judgment is the orchestrator's, never the builder's; everything unmarked runs Sonnet; stamp the map's frontier alias per `CLAUDE.md` § Model Selection (durable default `opus`). Write the map into STATE.md TOP.
5. **Wake-sensor** — launch detached, once per train: `nohup .claude/scripts/wave-sensor.sh {builder} {your-handle} >/dev/null 2>&1 &` (tick 10m/4.5m; states + control files: § O4); verify `tmp/wave-sensor/{builder}.heartbeat` appears within one tick.

## O2 — Reconcile (replaces plan/architecture)

Fan out ONE read-only reconciler per routed project against `$WT` (`Explore`, Sonnet; parallel; boundary prep runs them against post-merge `main` — § O6.3). Each verifies its project's portion of the spec against live code and returns binding, reality-wins corrections:

- **Anchors + file plan** — every Named anchor, path, symbol, and signature the spec cites exists as described; count call sites the spec enumerates (an undercount ships a compile failure); a path cited as a DATA SOURCE must CONTAIN the claimed data, not merely exist; every to-DELETE file grep-checked single-purpose — kept symbols convert it to `EDIT (strip X)` by def-boundaries; a deleted shared symbol/field/state-key's coupling enumerated tree-wide — readers AND every test literal; under-cited scope = binding correction.
- **Cross-system literals** — every literal the spec introduces that joins a live registry or convention (metric names, {QUEUE} message types, audit-map keys, enum values, i18n keys): grep the live convention; flag divergence.
- **Claims panel** — walker MANIFEST-VERIFY (`scriptPath:'.claude/workflows/wave-walker.js'`, `args.manifestPath`) probes every load-bearing claim refute-first — behavioral derivations, premises (anchor existence is not enough) — and flags cross-task conflicts + freeloader tasks; its REFUTED/PARTIAL verdicts and conflicts fold into the corrections.
- **Persist hops** — every new persisted field's write path AND read-back path named; a missing reader is a binding correction.

Write the merged result to `$WAVES/{wave-name}/manifest-corrections.md`. The spec + corrections are the build truth; a correction contradicting founder intent goes through the SPEC-CONFLICT rule instead.

## O3 — Brief & launch

Write `$WAVES/{wave-name}/BRIEF.md` for the builder:

- **Identity & env** — worktree path, branch, ports, orchestrator chat name; test stack is builder-owned: `make -C $WT/{INFRA_PROJECT} nuke-test-pipeline PIPELINE={wave-name}` → `up-test-pipeline` → `db-setup-test-template-pipeline` → `db-setup-test-pipeline` → `health-test-pipeline`; stand-up and teardown are the builder's (§ O6.3 verify).
- **Build truth** — pointers to `manifest.md` + `manifest-corrections.md` + the task map. Implement tasks in order; the per-task `**Build agents:**` line names when to spawn db-admin (`{INFRA_PROJECT}/.claude/agents/db-admin.md`) or ui-ux (`{project-ui}/.claude/agents/ui-ux.md`); the code is written by the builder's Sonnet dev hands (its own turns coordinate and judge — wave/builder.md § Orchestrated mode).
- **Task loop + guard rails** — canonical home: `wave/builder.md` § Orchestrated mode (task loop, END-YOUR-TURN, checkpoint discipline, ack echo, SPEC-CONFLICT escalation — the injected command carries it; a protocol change edits that file, never a restatement here); the BRIEF carries only this wave's parameters.

Launch — the **train goal fires ONCE, at train start** (per `/chat:goal` § Launch mechanics — the orchestrator authors the string itself, no interrogation):

1. **Train start only:** `chat.sh inject --no-sig {builder} "/goal deliver the {train-name} wave train — {N} waves per the Train-stamped manifests under $WAVES (one /wave:builder BRIEF per wave, /wave:builder Orchestrated mode); act ONLY on injected turns — a task verdict, a /wave:builder BRIEF, or a boundary brief; awaiting one IS goal progress: end the turn empty-handed, never start work unprompted, never open $WAVES manifests yourself, never run /jc unprompted, never self-schedule timers or background waiters — deferred Action Items are the orchestrator's to dispatch; the train ends when the orchestrator announces the train complete or terminated"` — ≤4k, single line; capture-verify (`Goal set` + the `◎ /goal active` statusline mark). Persist the string byte-identical to `$WAVES/{first-wave}/train-goal.md` — the ONLY durable copy; every re-fire reads it.
2. `/wave:builder $WAVES/{wave-name}/BRIEF.md — God speed.` — capture-verify delivery (exit-3 or typed-not-submitted → re-inject `--force-now`).

**Mid-train wave launches are a SINGLE inject** — capture the builder first and confirm `◎ /goal active` still shows (gone → re-fire + verify per § O4 GOAL-DROPPED, THEN the BRIEF); a fresh per-wave `/goal` is FORBIDDEN mid-train — it would REPLACE the train goal. A respawned builder (dead chat) gets the same re-fire-then-brief sequence before anything else. Verify the wake-sensor holds (§ O4 Liveness), plus the external heartbeat — a host cron that alerts the founder when no `$WAVES/*/STATE.md` was touched >45 min (glob, never a fixed path).

## O4 — Verify loop

**Ping-driven (fast path).** A builder ping (`Task #{N} done — report: {abs path}`) fires a turn immediately:

1. Read the task report + the task's **diff card**: an `Explore` (Sonnet) reader runs `git -C $WT diff {checkpoint N-1}..{checkpoint N}` and returns files, symbols, contracts touched, anomalies vs the task section. Read the FULL diff yourself only for `frontier-hands:`/{DOMAIN_ADJ} tasks and every ~5th task (sampled audit) — hygiene, GATE-1, and the walker re-cover every line.
2. Check against the spec task sections + corrections — Expected/Got on every gap; reject any changed path outside `$WT`/`$TASKS`.
3. Relay the builder's checkpoint request to gitter (Phase: WORKTREE-CHECKPOINT, explicit file list) when the builder hasn't already had it taken.
4. Verdict inject, one line: `T{N} verdict: proceed [frontier-hands: {spans}]` | `T{N} verdict: fix — {items}` | `proceed-disjoint T{M}` (parks a question while a disjoint task runs). Rule `frontier-request` pings in the verdict — grant by cost-of-being-wrong. Every verdict carries the task id; the builder's next ping echoes it (ack). Unacknowledged verdict at next contact → re-inject `--force-now`, confirm via capture. For a provably disjoint, all-Sonnet-hands mechanical span, one verdict may pre-authorize it — `T{a}-{b} verdict: proceed-span` — the span's cards get one combined review at T{b}, any finding there converting to `fix` verdicts.
5. STATE.md: rewrite TOP + `## Next`, append the outcome line below the marker. Update report.md's task row; append any fired retro entry (§ Steering conscience) before the turn ends.

**Wake-sensor (watchdog).** The periodic watch runs OFF this chat in the detached collector script `.claude/scripts/wave-sensor.sh` (O1 item 5; regex-first pane classification; tick log + heartbeat under `tmp/wave-sensor/`). The sensor stays silent on clear ticks — it classifies, NEVER rules; an actionable state lands ONE inject `sensor {state(s)}: {statusline}`. On it, re-capture fresh (`/chat:capture {builder}`) and act — the payload routes, never rules:

- GOAL-DROPPED → re-fire from `train-goal.md` byte-identical before anything else.
- IDLE / FINISHED-TASK → never re-sleep: a verdict is owed, the next step stalled (inspect `$TASKS` + `git -C $WT status`, corrective inject), or awaiting-agents — the benign idle: `echo {n} > tmp/wave-sensor/{builder}.snooze`. In `BOUNDARY-ACTIVE`: a ruling is owed (§ O6.2 sweep now) or the lane stalled (re-inject the boundary brief). In `idle SANCTIONED` (§ O6.4 flips the mode file): a WORKING builder is the incident — halt it, log `friction`.
- FROZEN → the shell may have EXITED: check `ps` + the run-log's `EXIT=` line; exited → wake-inject the result (finalize, never re-run).
- LONG-TURN → judge by the thrash signature (tmux-Escape lever).
- CTX → queue a `/compact` for the next task boundary, injected BEFORE the pending verdict, never mid-task. EVERY builder `/compact` — any phase — carries a FOCUS in front (position + immediate-next + STATE.md pointer + Orchestrated-mode reminder), never bare, and is a PLAIN inject (never `--then`, it jams un-submitted) + capture-verify `◎` (gone → re-fire); the FIRST post-compact inject starts `/wave:builder {BRIEF path}` (verdict/steer rides after).
- LIMITS (statusline gauges `5h-used:NN% ↻…`/`7d-used:NN% ↻…` — 5-hour/7-day account windows, % USED, `↻` = resets-in; one account: builder gauges = yours; either ≥80%) → **GENTLE PAUSE** — wrap in-flight work, start nothing new, park at a safe point (current item → checkpoint, builder idle), write the pickup point to STATE.md, stop the sensor + any fallback loop, arm a one-shot resume cron (CronCreate recurring:false) at reset+10m that relaunches the sensor and resumes from STATE.
- DEAD → respawn per § Resume.

**Liveness:** sensor armed → every acting turn stats its heartbeat (stale >25 min → relaunch via O1 item 5's line). NO sensor running (mid-wave protocol reload, relaunch failing) → the legacy self-wake stands: `/loop 10m` — env unset → `4.5m` (delete any prior loop id, create fresh — never two) — each wake captures the builder and runs these branches yourself; a non-acting fallback wake (capture → re-sleep) narrates ≤3 lines; stand it down once a sensor holds.

**tmux-Escape lever.** To land a time-critical correction or break a runaway-dispatch turn: `tmux -S /tmp/tmux-$(id -u)/{builder-session} send-keys Escape` → the queued steer processes. Escape breaks only an ACTIVE turn (no-op on idle async-wait) and never kills a background sub-agent — the builder kills its own on your steer. Intervene on the thrash SIGNATURE (zero files landed + climbing tokens across captures), never on dispatch category — an exact-briefed producing hand finishes. **Modal deadlock** (a permission modal swallows the statusline; label-resolve breaks): `chat.sh modal {builder-session} deny {N}` — DENY-ONLY, only when the choice is unambiguous and the denied action is the destructive one, never blind-accept; capture-verify; log `friction`.

**Wake discriminator** (a wake finding no thread, or any fresh resume — rule on artifacts in this order, never on memory; every branch is scoped to THIS train — the `**Train:**` stamp matched against the source path carried in the invocation, steer, or lock payload; a lock collision on a DIFFERENT train's wave is a hard diagnostic stop): (0) this train's source still holds `## Wave` headings whose Train-stamped manifests already exist → the split crashed mid-flight: complete the split + source reset FIRST. (1) a wave dir whose STATE.md stamp is an unfinished boundary (`BOUNDARY-ACTIVE`/`GREEN`, no `CLOSED`) → `/chat:capture {builder}` FIRST — busy on boundary duties → ADOPT: read artifact freshness, rule on unruled findings, wait; redo ONLY what the capture + artifact mtimes prove dead (the recorded walker run-id recovers its ledger — never relaunch blind); then launch + close. (2) a staged wave (`STAGED`/`CUT`, not `LAUNCHED`) → finish its launch from the stamp (◎-verify → single BRIEF inject — never a fresh per-wave `/goal`). (3) an in-flight wave (worktree + `LAUNCHED`) → re-enter O4 on it, first sweeping `$TASKS` for pings that arrived unanswered. (4) no wave dir claims the thread AND un-run Train-stamped manifests of THIS train exist → stage the lowest `## Wave {k}` of them (§ O6.3 seed-only) and continue. (5) source not yet split → before re-invoking `/wave:orchestrator {builder} {source}`, grep `tmp/dev/archive/waves/*/manifest.md` for a `**Train:**` stamp naming this source — a match = the train already ran: HALT, never re-split.

**Context discipline.** Read task reports and diff cards only — never raw test output or file dumps; bulky verification goes to disposable Sonnet/Haiku readers. Pre-ruling claim checks: wave-walker VERIFY mode (`scriptPath:'.claude/workflows/wave-walker.js'`, `args.claims`/`args.manifestPath` — name-lookup serves stale); any ad-hoc Workflow `agent()` inherits YOUR model — pass `model:'sonnet', effort:'xhigh'` (task-map frontier alias per frontier-hands logic). Past ~45% context, self-compact at the next natural pause (hold: wave position, verdicts owed, escalations, sensor state) — ALWAYS with the § O6.6 `--then` steer (`/wave:orchestrator {builder} resume {wave-name}`).

**Milestones** (marked in the task map): before the next task starts — cross-project typecheck + affected integration profiles in the worktree; a reconcile refresh over ONLY the anchors the SYNC merge diff touched; **gitter SYNC** (Phase: SYNC — merge current `main` INTO `pipeline/{wave-name}` inside the worktree) so divergence surfaces under the wave's tests, never in a blind end-merge.

**Steering conscience (`retro.md`).** The orchestrator audits its own steering: append to `$WAVES/{wave-name}/retro.md` — any phase, at the moment of detection, never reconstructed at close-out. reversal and friction are mechanical: the event fires the entry, no judgment call. intent-gap, tier, and obs are judgment: resolve every doubt toward logging. A qualifying event without an entry is itself a steering failure. Entry:

```
## R{n} · {T{N} | phase} · {reversal | intent-gap | friction | tier | obs}
Expected: {what the steer assumed}
Got: {what happened}
Amend: {file § section — concrete change; or `judgment` when no text fix exists}
```

- reversal: a fix lands in a task already verdicted `proceed`, whatever surfaced it
- intent-gap: a goal was met as written yet needed a re-steer to be right — the amending verdict/brief line is the event
- friction: machinery burned time/tokens — unacked verdict, re-inject, deadlock, respawn, resume confusion, verdict latency leaving the builder idle
- tier: a `frontier-hands` grant plain Sonnet work satisfied, or a Sonnet-hand defect a frontier grant would have prevented
- obs: any other framework improvement worth making next time — including a wave O0 should have bounced to `/wave:refine`

A builder `retro:` ping note becomes its own entry (note verbatim in `Got:`). Consumer: the founder's `/pcm` pass over archived retros, reconciled against co-archived artifacts — a post-`proceed` fix with no reversal entry is a withheld entry. A folded amend is stamped `Resolved: {date} — {where}` under its entry, in place.

## O5 — End-of-wave (order is load-bearing: hygiene remediation converges BEFORE the full gate)

Builder over ~50% context at O5 entry → inject a focused `/compact` (hold: O5 duties, task-map tail, the train goal) BEFORE hygiene — hygiene+GATE-1 is the wave's heaviest context phase.

1. **SYNC** — gitter merges current `main` into the branch one final time.
2. **Hygiene review** — spawn Opus: execute `audit/code-hygiene.md` scope `diff` over `git -C $WT diff --name-only main...pipeline/{wave-name}` → `$WAVES/{wave-name}/hygiene-review.md` with a verdict line.
3. **Remediation** — builder fixes every hygiene code finding in the worktree (fix → targeted re-test → re-audit the touched category); loop until clean, cap 2 rounds; residuals land in the report and `### /jc Action Items`.
4. **GATE-1 (pre-merge full — LAST pre-merge step)** — builder-executed per `wave/builder.md` § End-of-wave duties; verdicts land in `$WAVES/{wave-name}/gate1.md` — a skipped tier (`envBlocked`/`INTEGRATION-UNRUN`) is never green; escalation → BLOCKED-DEFERRED, worktree preserved.
5. **MERGE** — gitter, `Wave-mode: v2` (precondition = `gate1.md` all green AND gate-COMPLETE: its verdicts cover every top-level dir in `git diff --name-only main...pipeline/{wave-name}` — a touched project without one re-runs GATE-1 for it; `$DOCS` derives to `$WAVES/{wave-name}`), serialized by `git-lock.sh`. Main moved since SYNC → gitter re-syncs and the builder re-runs affected profiles only before retrying. When gitter RETURNS the merge SHA (never on entering MERGE): write `**Merge SHA:** {sha}` into report.md, capture the builder — context over 70% → inject a FOCUSED pre-boundary `/compact` first (hold: boundary duties, artifact paths, /jc rules, the train goal + allow-list) — then inject the **boundary brief** (one line; switches the builder to BOUNDARY mode, `wave/builder.md` § Boundary duties): `Boundary {wave-name} — merge SHA in report.md; run § Boundary duties, ping each artifact.` Stamp STATE.md TOP `BOUNDARY-ACTIVE — builder executing boundary duties`. The train goal stays live — nothing closes it mid-train.

## O6 — Boundary (builder executes, orchestrator observes and steers)

Every transition rewrites the closing wave's STATE.md TOP stamp (`BOUNDARY-ACTIVE → GREEN → CLOSED`; the staged wave's own STATE.md runs `STAGED → CUT → LAUNCHED`) at the moment it happens — the stamps are the wake discriminator's and the heartbeat's substrate. Every verifier artifact persists the moment its verifier returns, never batched into a later check. **OVERLAPPED BOUNDARY (founder-ratified option):** at MERGE, SETUP cuts the next worktree from the MERGE SHA immediately (not waiting green); GATE-2 + walker run as background lanes while the builder starts next-wave first tasks; a BLOCKING finding HALTS next-wave tasks until remediated on `main` + gitter SYNC into the new branch; re-cut ONLY if the merge itself reverts.

1. **Verify on `main` (builder-executed)** — the builder runs `wave/builder.md` § Boundary duties end-to-end: background walker (merge-SHA mode off report.md's `**Merge SHA:**` line, run-id recorded in STATE.md at launch) concurrent with foreground-serial GATE-2 on the canonical stack. Observe the artifacts: per-project `gate2.md` verdicts as they land, `walker-ledger.json` on return, a ping each.
2. **Findings lane (orchestrator rules, builder implements)** — severity floor: a GATE-2 regression, a BROKEN thread, or a confirmed high+ anomaly **blocks the boundary**; MINOR and UNPROVEN findings land in `### /jc Action Items` and never delay the cut. Each ruling is a verdict inject with an id (ack law: § O4); **green-detection is POLL-based** — every acting turn sweeps `$WAVES/{wave-name}/` for unruled gate2 rows/ledger findings and rules regardless of ping delivery. Fix-now rulings the builder implements via `/jc` (timing per its § Boundary duties); every fix-now fix — including orchestrator-designed — gets one independent judge over its diff before it counts fixed (QA-green ≠ convention-correct); a fix in project P re-runs P's boundary reconciler (§ O6.3). gate2.md records fix SHAs + re-run scope. Cap 2 rounds; residuals defer, named in the announce. **SHIPWRECK or a confirmed security (R6/R7) anomaly is stop-the-train**, sequenced: orchestrator halt-inject → builder halt-ack names its dirty files (its § Boundary duties) → gitter REVERT (stash exactly the named dirty files, `git revert -m 1 {merge-sha}`; branch preserved — the wave re-enters O5 remediation) or, where a revert is worse, hard-block + founder escalation — and EVERY train-ending abnormal path (stop-the-train, terminal BLOCKED-DEFERRED, founder halt) fires the goal-closing announce (`Train terminated.`) + stand-down before anything else stops.

## Resume

- **Builder died** — DEAD escalation → respawn `--detach`, re-fire the train goal (§ O4 GOAL-DROPPED; goal state is per-session), THEN the RESUME brief: `git -C $WT diff {last-checkpoint}..` to see half-done work, hard-reset to the last checkpoint, redo the in-flight task from clean — never continue blind (boundary-mode death: no worktree — resume § Boundary duties from artifacts). STATE.md + checkpoints + task reports are the durable index.
- **Orchestrator died** — the external heartbeat alerts the founder; a fresh `/wave:orchestrator {builder} resume {wave-name}` finds the singleton lock, adopts it (stale PID), reads STATE.md TOP, captures the builder, and re-enters O4 at the recorded position. On ANY orchestrator resume — fresh invocation or a relaunch of this chat: re-run `/chat:whoami` + rewrite STATE.md's orchestrator handle/ping fast-path; kill + relaunch the sensor with the fresh handle (its inject target is the dead one; a fallback self-loop, if armed: re-arm DELETE-then-create); re-stamp the singleton lock PID; inject the builder one status no-op so its reply-footer refreshes your handle.
- **Boundary death** (self-compact steer swallowed, or a crash anywhere in § O6) — rule by the O4 **Wake discriminator**, on artifacts in its stated order: unfinished-boundary wave first, staged wave second, in-flight wave third, source re-invoke LAST and only when no wave dir claims the thread. Mid-wave death is NEVER the source form — STATE.md present → `resume {wave-name}`; the in-flight partition lives only in that wave's manifest, not the source.
- **BLOCKED-DEFERRED** — worktree + ports preserved; `$WAVES/{wave-name}/BLOCKED.md` carries the trigger + resume protocol (template: `docs/commands/build/references/build-reference.md` § BLOCKED.md).

## Rules

- **NEVER delegate Skill calls to sub-agents** — invoke directly.
- **Observe and steer, never execute** — ALL real work (code, gates, walker, `/jc` fixes, teardown) is the builder's; the orchestrator reads artifacts, rules findings, stages documents, and drives gitter.
- **Only gitter runs git** — checkpoints, SYNCs, merges, doc commits all through it.
- **Every gate verdict is a persisted artifact** (`gate1.md`, `gate2.md`, `hygiene-review.md`, `walker-ledger.json`) — a verdict that exists only in chat context does not exist.
- **Verdict-before-next-task** — the builder proceeds only on an acked `proceed`; the orchestrator reviews every task diff, no rubber stamps.
- **Hygiene runs pre-merge; GATE-1 runs after its remediation converges** — nothing merges unexercised by a full suite. The walker runs post-merge on `main`, concurrent with GATE-2; every blocking finding is fixed there via `/jc` before the next worktree is cut.
- **STATE.md discipline** — TOP rewritten only at lifecycle transitions; below the marker append-only via shell append (`cat >>`), never Edit-tool rewrites (Edit makes the whole file context-resident).
- **Founder amendments mid-wave** land only in NOT-yet-started tasks; a running task is never re-scoped.
- **Founder PAUSE/HOLD** — `/goal clear` the builder FIRST (an active goal keeps it churning) AND stop the sensor + any fallback loop — zero wakes until the founder's RESUME: relaunch the sensor, re-fire the goal from `train-goal.md` byte-identical.
