# v0.21.0 — 2026-06-10

The turn-done desktop notification grows up — it now tells you which project, which session, what you last asked, and how long it ran — and a concurrency race in its turn timer is fixed. Two `CLAUDE.md` refinements ride along.

## Added

- Scripts: `notify.sh` — the Stop-hook banner carries full context: the project name (title + body lead), the human session name when the harness exposes one (set via `/rename`, read from `$CLAUDE_CONFIG_DIR/sessions/*.json`), the last typed prompt for at-a-glance "which chat just finished", and a humanized turn duration (`45s` · `2m13s` · `1h02m`). All additive and degrade gracefully: a single-account install with no named sessions falls back to the 8-char session id, and the last-prompt snippet is `jq`-gated (omitted without `jq`).
- Tier A: `CLAUDE.md` — new Meta non-negotiable "When in doubt, do the right thing": on genuinely ambiguous decisions choose the correct path over the convenient one, even at the price of re-architecting; think twice, do once.

## Changed

- Scripts: `notify.sh` — banner engine is now **osascript-first**, with `terminal-notifier` only as a fallback. osascript ships with every macOS; terminal-notifier needs a separate install and its older builds silently drop the banner on current macOS while still chiming ("sound but no banner"). `msg`/`title` are AppleScript-escaped now that the body carries a user-set name.
- Scripts: `notify.sh` — the turn-timing stamp is keyed per `session_id` (`/tmp/{CHARACTER_NAME_LOWER}_turn_start.<id>`) instead of one shared path, fixing a race where concurrent sessions of the same user mis-timed or silenced each other's notifications. The PreToolUse `start` branch reads `session_id` from stdin to build the key; an empty id falls back to the bare path.
- Tier A: `CLAUDE.md` — removed the Request Routing table; routing now lives in each command's `description:` frontmatter (the harness injects that registry), ending the table/registry duplication, and the skill-loading note is compressed to one mandatory-load line. The per-command frontmatter migration and `pcm.md` audit-scope rewrite are a follow-up release.
