# v0.27.0 — 2026-06-12

## Added

- Tier A: documenter — new **EPIC mode** (`/documenter epic {epic-name?}`): consolidates the current session's work straight into the active epic, inline — the conversation is the source, so there is no intermediate dump file and no duplicate manifest write. Replaces `/save epic`.
- Tier A: documenter — **Epic consolidation contract**: the single rule set governing every epic write (documenter ARCHIVE Step 2j, EPIC mode, wave Step 3.5). `update.md` becomes the epic's work doc — `## State of work` (rewritten every consolidation: what is live, in-flight position, ordered next steps) + `## Delivered` (per-area current-state, merged, superseded designs rewritten away). Manifest `## Progress Log` shrinks to one line per milestone; `## Key Decisions`/`## Discoveries` are deduped merges; new `## Files` section indexes topic files for selective loading.

## Changed

- Tier A: CLAUDE.md § Epics — **epic read/update redesigned as current-state, never append-only logs.** Structure: `manifest.md` (anchor) + `update.md` (work doc) + topic files indexed in the manifest's `## Files` + `archive/` cold storage. Load protocol is constant-cost: read `manifest.md` + `update.md`, open topic files from `## Files` only as the task requires, never `archive/`. `templates/epics/TEMPLATE.md` rewritten to the new design.
