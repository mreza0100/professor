# v0.22.0 — 2026-06-10

The registry-over-tables migration completes (every command template now carries its routing in frontmatter, and `/pcm` audits the registry instead of tables), the documentation scaffold ships so no adopter inherits dangling hub references, the framework learns that standalone skills release from their own repos, and `p:refine`'s approval gate stops conflating spec approval with execution.

## Added

- Mechanics: `templates/docs-agents/` — new shipped scaffold: an `_index.md` hub skeleton and a `standards.md` skeleton (architecture source-of-truth with the "every architect reads this before designing" contract). SETUP gains **Phase 2.7**: seed `docs/agents/` from the skeletons, then run `/documenter bootstrap` to build the clusters from code — or defer with marked rows when the project is too new, mirroring empty-skill hydration. Closes the gap where `/documenter` and the architects referenced a hub, `standards.md`, and clusters that never existed on the adopter's disk.
- Mechanics: `p:blueprint` — release step **5b**: a `release.md` entry naming a `sources.json` skill is an informational version-bump line only (the blueprint never vendors those skills); the maintainer is reminded to push the change to the skill's own repo and cut a release there. Update gains **Step 8b**: check each `sources.json` repo for a version newer than the installed skill's `version:` frontmatter and offer a re-fetch — never downgrade a skill whose installed version is ahead (that marks an unreleased local fix).
- Tier A: `pcm.md` — ledger rule "Standalone skills are a special case": a change to a `sources.json` skill bumps its `version:` frontmatter, logs a version-bump-only Professor entry, and routes the substance to the skill's own repo.

## Changed

- Mechanics: all 14 command templates now carry `name:` + `description:` (+ `argument-hint`) frontmatter — the routing signal the harness injects into each session. Descriptions are parameterized per archetype ({PROJECT_NAME}, {MARKET_SEGMENT}, {REGULATION}, {USER_PERSONA}, {AI_PROJECT}); previously only `save.md` had frontmatter. (safe-auto)
- Tier A: `pcm.md` — registry-over-tables rewiring to match v0.21.0's CLAUDE.md: invariant "Tables match files" replaced by "Registry over tables"; the `agents`/`commands`/`skills`/`cross-refs` audit scopes check frontmatter coverage and the Agents inventory instead of table sync; Special Operations write frontmatter instead of CLAUDE.md table rows.
- Tier A: `p:refine` — the R4 gate approves the **wave.md spec** and ends there; running `/wave` is the founder's separate decision, after and apart from the gate. The closing "Run `/wave` to execute" instruction is dropped.
- Docs: `refresh.md` — §1 mines the `docs/agents/` scaffold (hub format + standards skeleton + cluster convention; structure only, never doc content) and §3 ships `templates/docs-agents/`.

## Migration

#### For: adopters with a customized local `pcm.md` or `p:refine`

The `pcm.md` audit-scope and `p:refine` R4 changes land as `A→B→C` conflicts if you customized those files — review the diff and merge the registry-over-tables scopes and the spec-only R4 wording into your copies; both are wording/protocol changes with no structural moves.

#### For: all adopters

After updating, seed `docs/agents/` if you don't already have it: copy `templates/docs-agents/{_index.md,standards.md}` in, substitute your project tokens, then run `/documenter bootstrap` (or mark the cluster rows deferred until the codebase exists).
