# v0.23.0 — 2026-06-10

The framework learns to finish marathons: `/slow-burn` paces long-running work so the rolling session limit never cuts it short, and checkpoints so even a hard cutoff loses nothing.

## Added

- Mechanics: `/slow-burn` command (`templates/commands/slow-burn.md`) — session-limit pacing for long-running work. The session limit is token burn per rolling window, so the command splits the task into checkpointed rounds (`tmp/slow-burn/{slug}/state.md`, cross-session resume via `/slow-burn resume {slug}`), throttles with cache-aware pauses in two legal denominations (4-min naps keep the prompt cache warm; 25-min hibernations amortize one cache miss — 5–10-minute pauses pay the miss without buying throttle), scales with an intensity dial 0–10 (`/slow-burn N` mid-run; default 4; 0 removes pacing), cuts per-round burn with a token diet (sequential subagents, ranged reads, haiku downshift for mechanical work), escalates intensity under limit pressure, and narrates one heartbeat line per round. Fully generic — zero placeholders, ships as-is. (safe-auto)

## Changed

- Docs: README, BLUEPRINT.md, and the `p:blueprint` refresh reference rosters now include `/slow-burn` (Tier A command list, templates/commands listing). **`update`: skip — informational only.**
