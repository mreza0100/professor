# v0.29.0 — 2026-06-15

The `/chat` command family — cross-chat orchestration — plus the documenter ARCHIVE split, goal-definer → goal-manager with epic continuation prompts, gitter's scoped-commit discipline on shared `main`, and statusline effort/limit/ultracode upgrades.

## Added

- **Tier C: `/chat` command family — cross-chat orchestration.** Five new namespaced commands, all generic Claude-Code session plumbing (zero product domain), built on the existing `/chat:save` + the shared `chat-find.sh` resolver:
  - `/chat:find` — bare lookup primitive: resolves a pasted excerpt to a session across every account's transcript registry (current session excluded, hit-ranked), reporting session-id, transcript path, date range, and candidates. Underpins read/send/inject/capture. Script `chat-find.sh`.
  - `/chat:read` — read an earlier chat from a pasted excerpt: multi-account resolve via `chat-find.sh`, extract the full visible conversation to `tmp/chat-loads/{session-id}.md`, then read it with a 6-point LLM-optimized protocol (mental-model-first, tool-trails-are-pointers, low-resolution compaction summaries, distrust-stale-facts) and resume. Script `chat-read.sh` (the rename of `chat-load.sh`).
  - `/chat:send` — cross-chat mailbox bus: drops a message into a target chat's on-disk inbox; `inbox` subcommand reads this chat's own unread. Async (target reads on next resume). Scripts `chat-mail.sh`, `chat-mail-drain.sh` (UserPromptSubmit drain hook), `chat-mail-notify.sh` (SessionStart register+notify hook).
  - `/chat:inject` — forced-turn injector: auto-picks delivery — a live tmux pane gets send-keys typed and submitted now; a dormant chat gets the turn appended to its transcript, answered on resume. Scripts `chat-inject.sh`, `chat-nudge.sh` (delayed-discrete-Enter send-keys, works around bracketed-paste swallowing the newline).
  - `/chat:capture` — live-window snapshot: captures another chat's live tmux window (footer, spinner/idle state, task list, statusline) via `chat-capture.sh`; live-pane only, routes to `/chat:read` for dormant chats.
- **Tier B: `/documenter:archive` — namespaced ARCHIVE-mode command** mono-documenter invokes from the build/wave pipeline; runs the full pipeline-doc merge into permanent docs + epic consolidation. Declares `allowed-tools: Bash(cat:*)`.
- **Tier B: documenter bang-cat shared fragments** — `_base.txt` (the p:quality:doc-load + verify-against-code + approval-gate preamble, doc clusters/hub, Owned Documents, Document Registry) and `_epic-consolidation-contract.txt` (update.md / manifest consolidation rules, Professor-owned boundaries), injected verbatim via `` !`cat …` `` into `/documenter:archive`. The fragments deliberately carry a leading underscore and live outside the command-registration path so they never register as phantom commands — a single-source-of-truth mechanism for prose shared across documenter modes.
- **Tier B: `/goal-manager epic {name}` subcommand** — writes/refreshes an epic continuation prompt at `docs/epics/{name}/prompts/resume-latest.md` (+ a dated archive) as a `# Report` narrative + a `# Prompt` the founder pastes into `/goal` to resume a stopped session exactly where it left off. Integrated into the `/documenter epic` checkpoint.

## Changed

- **Tier A: gitter — Scoped-commit discipline on shared `main`.** New BANNED row for `git add -A`/`.`/`-u` and `git commit -a` ON MAIN; a 5-step safe-commit ritual for every commit on `main` (JC-COMMIT, DOCS-COMMIT, PUSH): `git restore --staged .` → explicit-path `git add` → `git status --porcelain` verify → HEREDOC commit → `git show --stat` verify. Rationale: `main` is a shared working tree — concurrent sessions pre-stage files and the orchestrator fences off held WIP, so a sweep commit lands gated work unauthorized (a sacred-ground breach). Phase-2 MERGE is exempt (a `pipeline/` branch is an isolated worktree).
- **Tier A: `/goal-manager` — long-run `/goal` hardening.** Step-0 written mental model the run forms before any work; resume/`/compact` re-forms the model before reading "the next step"; an autonomy clause (chain green milestones, surface only at decision gates); continuation prompts cite fast-moving facts by live source, never hardcoded counts; a phased-endgame gate (done = the success criteria, not a green slice and not a `/goal` banner); the settable prompt's first line is `/goal` and it stays within the input ceiling, bootstrap-linking a larger constitution file when the charter would overflow.
- **Tier C: statusline — effort / limit / ultracode upgrades** (`~/.claude/statusline-command.sh`). Effort gem-ramp 🔹low → 🔶medium → 💠high → 💎xhigh → 👑max (💤 when extended-thinking is off); ultracode 🚀 badge gated on a session-keyed marker; a 7-day rate-limit bar (the field was parsed but never rendered); a cache-hit % segment; a session-name 🔖 segment; a `COLUMNS` responsive width-gate that drops low-priority segments under 100 columns. The per-account medal badge ships **opt-in and off by default** (`ACCOUNT_BADGE=0`) with empty `ACCOUNT_EMAIL_1/2/3` slots the adopter fills with their own accounts.

## Removed

- **`/chat:load` retired** — superseded by `/chat:read` (script `chat-load.sh` → `chat-read.sh`). See Migration.
- **`/goal-definer` retired** — superseded by `/goal-manager`. See Migration.

## Migration

#### → For adopters with an installed blueprint:

- **`/chat:load` → `/chat:read`.** `/pcm update` places `chat/read.md` + `chat-read.sh` and removes `chat/load.md` + `chat-load.sh`. The new command is a superset (multi-account resolve + the 6-point read protocol). Re-point any `/chat:load` muscle memory and any contract/check-script command list (`chat/load` → `chat/read`). The `transcript-extract.jq` comment reference updates with it.
- **`/goal-definer` → `/goal-manager`.** `/pcm update` places `goal-manager.md` and removes `goal-definer.md`. Default compile mode is unchanged; the new `epic {name}` subcommand and the long-run hardening come with it. Update any command list (`goal-definer` → `goal-manager`).
- **New `/chat:*` commands and scripts.** `/pcm update` adds `chat/{find,read,send,inject,capture}.md` and the scripts `chat-find.sh`, `chat-read.sh`, `chat-inject.sh`, `chat-mail.sh`, `chat-mail-drain.sh`, `chat-mail-notify.sh`, `chat-nudge.sh`, `chat-capture.sh` (they require `tmux` for the live-pane paths and `jq`). If your install registers commands in a contract/check script, add `chat/find`, `chat/read`, `chat/send`, `chat/inject`, `chat/capture`, and `goal-manager`.
- **`/documenter:archive` + fragments.** `/pcm update` adds `commands/documenter/archive.md` and the two non-registered fragments `_base.txt` + `_epic-consolidation-contract.txt`. The `allowed-tools: Bash(cat:*)` declaration is required for the bang-cat injection.
