# v0.35.0 — 2026-06-22

The wave engine's test infra flips from per-agent-owned to **orchestrator-owned shared-stack**: the wave-build orchestrator stands one isolated test stack up once before Develop and tears it down once in a `finally`, while agents share the container and isolate via per-project worker DBs and queue segments. The implementer role drops to a Sonnet (spec-following) tier with judgment roles held on Opus; the fix loop gains a `pre-existing-orthogonal` short-circuit; a new wave-level GATE runs one full suite on integrated `main` after all pipelines merge; and a clutch of correctness fixes lands (false-green test output, Setup→Plan stage order, `.env.test` multi-service coherence). `/officer` is reframed as **our counsel** — advocate-not-referee, bounded by an honesty floor — and the blueprint `CLAUDE.md` gains the model-selection ladder and the "God speed" autonomy rule.

## Changed

- **Mechanics: TEST_INFRA contract flipped to shared-stack / orchestrator-owned (`workflows/wave-build.js` + `commands/wave/build.md`).** Replaces the per-agent up/down/nuke "sole occupant by construction" model: the orchestrator owns the stack lifecycle; agents SHARE one container and NEVER run `up-`/`down-`/`nuke-test-pipeline` (idempotent `db-setup-template`/`db-setup` allowed), isolating instead via a dedicated per-project worker DB (`{project}_test_{worker}`) and per-project queue segments (`{base}-{project}-{worker}`). Rewrites the Step 7 QA spawn brief, the Fix Loop infra paragraph, the GATE-1 discipline line, the GATE-2 spawn brief, and the wave-mode infra delta to match.
- **Mechanics: implementer model tier Opus → Sonnet (`workflows/wave-build.js` + `commands/wave/build.md`).** `devAgent` moves to the spec-following tier across Step 6 Develop, the Fix Loop developer step, and the Code-Review-apply developer spawn; judgment roles (planner, architect, ui-ux, db-admin, review, QA) stay Opus. Single-source pointer to `agent-models.md` added.
- **Mechanics: UNRUN-ENV ≠ green (`workflows/wave-build.js` + `commands/wave/build.md`).** New `envBlocked`/`INTEGRATION-UNRUN` outcome on GATE-1 (never green/PASS — carried to the wave-level gate) and on GATE-2 (distinct from a code failure); `envBlocked` added to the QA + PMQA schemas and the `qaRound`/PMQA briefs.
- **Scripts: `scripts/worktree.sh` — `.env.test` multi-service coherence.** The `{ENV_FILE_PROVISION}` rewrite now retargets EVERY test-port reference (DB URL, `DB_PORT`, and ALL object-store/queue service endpoints) to the pipeline-isolated stack, not just the DB URL — a missed rewrite silently pins the project (and any sibling reading the same file) onto the shared stack → cross-pipeline collisions. The rule is `^`-anchored (never touches a cross-service URL/HOST/comment), password-agnostic (matches a credential-less `user@host` DB URL — the cortex case), and slash-agnostic (matches endpoints with or without a trailing slash).
- **Docs: `commands/wave.md` + `commands/wave/build.md` — resume requires re-passing args.** Resume now requires the SAME args alongside `resumeFromRunId` (args aren't journaled → the args-guard throws), and documents the machine-crash mid-run recovery path (journal + worktrees + `main` survive → assess, relaunch with args + `resumeFromRunId`).
- **Docs: `docs-commands/build/references/build-reference.md` — Setup→Plan order + branched resume protocol.** Pipeline step map reordered so Git setup is step 1, Parallel analysis 2a, Consolidate plan 2b (was analysis/plan 1a/1b, Git setup 2); the § Audit-trail reference aligned to "Git Setup (Step 1)." The single linear `BLOCKED.md` resume protocol is rewritten into two origin-branched paths — A (defect pre-existing on `main` / orthogonal → fix on main + rebase or cherry-pick → re-spawn QA only) and B (defect in this pipeline's own uncommitted worktree work → fix in the worktree, no main detour, no rebase → re-spawn QA only) — with a shared converge tail; `pre-existing-orthogonal` added to the front of the trigger enum.

## Fixed

- **Scripts: `scripts/filter-test-output.sh` — empty result no longer reads as all-green.** A green run always emits an "N passed" summary (caught by the keep-grep), so an empty filtered result means the runner crashed/was killed (timeout/OOM) or printed an unrecognized format. The empty branch now surfaces the raw output tail (`tail -40`) under "(no pass/fail summary recognized — runner may have crashed or been killed; raw tail follows)" instead of the false "(all tests passed — verbose output suppressed)" message. Never silently swallows a failure as green again.
- **Mechanics: `agents/gitter.md` — Phase 1 SETUP Setup→Plan order drift.** SETUP is the FIRST pipeline stage (creates the worktree before planning/architecture run), not a post-plan stage: the precondition header is corrected from "Invoked **after** `1-plan.md` is written" to "First pipeline stage — creates the worktree before planning and architecture run," and the validate-preconditions check from "Confirm `1-plan.md` exists" to "Confirm `0-task.md` exists (the pre-placed task spec)."

## Migration

- **`settings.json` agent-teams default flipped 1 → 0.** `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS` now defaults to `"0"` so every fresh install starts with experimental agent-teams disabled. `/pcm:update` re-applies the flip; if you customized `settings.json` it surfaces as a both-changed conflict on that one key. **For adopters:** set `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS` to `"0"` in your `settings.json` `env` block (no other key touched); the change takes effect on the **next session restart**, not hot. After a machine crash mid-run (journal + worktrees + `main` survive), assess state, then relaunch with the original args + `resumeFromRunId` rather than starting fresh.
