# v0.31.0 — 2026-06-16

Refinements to the chat: inter-chat orchestration family — injected messages now carry a runnable reply command, `/chat:inject` accepts a simpler target-first syntax, `/chat:load` stops writing, and one subcommand is renamed for consistency.

## Added

- **Scripts: `commands/chat/chat.sh` (inject) — sender signature on every message.** Each injected message now ends with a footer — `— from {name} · sid {session-id} · to reply: /chat:inject {sender tmux} <message>` — so the recipient gets the source and a runnable reply command. The `sid` is always derived; the `to reply:` line appears when the sender is in tmux (its session is the live reply handle); the human `{name}` comes from `CHAT_INJECT_FROM_NAME`. The LIVE typed footer is single-line (a bare newline submits in the Claude Code input); the RESUME transcript and the long-message spill file get a readable block footer.
- **Docs: `commands/chat/inject.md` — target-first invocation.** `/chat:inject {target} {message}` (the form the reply footer teaches) now works alongside the legacy `{message} :: {target}` (kept for multi-word excerpt targets).

## Changed

- **Scripts + Docs: `commands/chat/chat.sh` + `load.md` — `/chat:load` is write-free.** It enumerates the file set and forces a full read of every file, writing nothing — the mental-model-doc rewrite is gone.
- **Scripts + Docs: `commands/chat/chat.sh` + `whoami.md` — subcommand `who-i-am` → `whoami`.** Unifies the engine subcommand with the `/chat:whoami` command name on the Unix idiom; `whoami.md` also drops the stale `:: {target}` example for the target-first form. The `/chat:whoami` command name is unchanged.

## Removed

- **Scripts: `commands/chat/chat.sh` — the `load-check` subcommand.** It verified the written mental-model doc; with `/chat:load` writing nothing, the gate has no purpose.

## Migration

#### For: anyone scripting `chat.sh` subcommands directly

`chat.sh who-i-am` → `chat.sh whoami`; `chat.sh load-check` is gone (and `load` no longer asks for a written doc). The slash commands (`/chat:whoami`, `/chat:load`, `/chat:inject`) keep their names; only `/chat:inject` gains the optional target-first syntax, with `{message} :: {target}` still accepted.
