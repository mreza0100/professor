# v0.30.0 — 2026-06-16

The chat: command family — inter-chat orchestration — consolidated to a single engine and reshaped to injection-only delivery.

## Changed

- **chat: family → one engine.** The eight `chat-*.sh` scripts plus `save-transcript.sh` and `transcript-extract.jq` collapse into a single `.claude/commands/chat/chat.sh` with subcommands (`find · read · inject · ls · capture · save · who-i-am · load · extract · tail`); the jq renderer is inlined. Every `chat/*.md` command now calls `chat.sh <sub>`.
- **`/chat:save` is now a mechanical transcript copy** straight from the session registry — no model rewrite. The model-written continuation briefing moved to the new `/chat:dump`.
- **`/chat:read` takes a last-N-lines limit** (`/chat:read {excerpt} N`); a bare number reads this chat's own last N lines.

## Added

- **`/chat:inject` auto-picks delivery:** `self` or a live tmux session → typed in and submitted now (send-keys); a session-id or pasted excerpt → appended to that chat's transcript, answered on resume. Messages over the wire cap spill to a file + a short pointer, so a long steer never jams the paste buffer.
- **`/chat:whoami`** — this chat's own tmux session (the handle other chats inject to).
- **`/chat:ls`** — the live chats whose cwd is in this repo, with idle/busy state and a screen snippet.
- **`/chat:load`** — force-load a directory or file set: enumerate every file, rewrite each into a mental-model doc, gated by a coverage + substance check so a lazy skim fails.
- **`/chat:dump`** — the model-written briefing-plus-transcript continuation save (the former `/chat:save`).

## Removed

- **`/chat:send` and the entire inbox/mailbox mechanism** (the chat-mail scripts and the drain/notify hooks). Inter-chat delivery is injection-only now — a pull-based inbox sat unread; injection forces the read.

## Breaking

- `/chat:send` no longer exists.

## Migration

#### For: anyone using `/chat:send`

Replace it with **`/chat:inject`** — target a live **tmux session name** for an immediate land (typed in + submitted), or a **session-id / pasted excerpt** for a resume-time transcript append. There is no inbox to poll.
