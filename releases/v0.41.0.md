# v0.41.0 — 2026-07-02

The `host-swap/` cc-fleet tools shipped in v0.40.0 catch up to the fleet logic actually
running upstream: `cc-ls` gains a live-agent attach view and auto-unhide, `/bb`'s close
sequence polls instead of guessing a sleep, and `cc-reap` learns two new KEEP guards. The
statusline template gains the breadcrumb write the fleet depends on (plus a latent bug
fix), and `INSTALL.md` picks up the multi-account command-mirror step the fleet needs to
be visible on every account.

## Changed

- **`host-swap/cc-ls.snippet.sh` — the ⚙ agent attach view + auto-unhide.** A live background/forked session (no tmux socket — `--resume` refuses it) now shows as `⚙ agent`; Enter opens the `claude agents --cwd` attach view under its owning account instead of failing to resume. Hidden chats auto-return when their transcript grows past a lazily-recorded size baseline — hiding is for finished chats, not a way to silence a still-working one. Also: tab-safe field parsing (an empty pane title no longer collapses columns), `${(q)}`-quoted interpolated paths in the two tmux launches, a resume failure-net that falls through to the agent view instead of dying `[exited]`, and the namecache bumped to `.v3`.
- **`host-swap/cc-hide.sh` — `/bb`'s close sequence polls instead of sleeping.** Was a fixed 4s sleep before `kill-pane`; now polls up to 20s (compaction can outlive a fixed grace) and records the post-exit transcript size as `cc-ls`'s auto-unhide baseline, so the `/exit` flush itself is never mistaken for new activity.
- **`host-swap/cc-reap.sh` — two new KEEP guards.** A `cc-new-*` detached teammate is kept as `mate` (its parent's `/bb` reaps it); any session `claude agents --json` reports **busy** is kept as `busy` (deliberately detached, still working) — scanned across the default account plus every extra `~/.claudeN` config dir found on disk.
- **`statusline/statusline-command.sh` — writes the fleet's `/tmp/cc-sid` breadcrumb.** Adds the `.transcript_path` → `/tmp/cc-sid/<socket>` write (`chmod 700` — the breadcrumbs are transcript paths, the name cache carries prompt text) that `cc-ls` needs to size and resolve each chat. A no-op outside tmux, so it's safe to leave in even without the fleet tools installed.

  #### → For: adopters who already installed `host-swap/` in v0.40.0 — re-copy `cc-ls.snippet.sh`, `cc-hide.sh`, `cc-reap.sh`, and (if using the Professor statusline) `statusline-command.sh`. All four are drop-in replacements; no account-table edits needed.

## Fixed

- **`statusline/statusline-command.sh` — the `jq` `join()` call was missing its separator.** The single `jq` pass reads 27 fields via `IFS=$'\x1f' read -r`, but the `jq` program joined them with `join("")` instead of the unit separator — every field silently collapsed into `$MODEL` and the rest of the statusline rendered blank (directory, git, cost, everything). Fixed by writing the unit-separator as a `jq` unicode escape (readable in source) instead of a raw invisible control byte.

## Docs

- **`INSTALL.md` — new Step 8.4a: host-swap + the account command-mirror.** Wires the opt-in multi-account fleet install into the main install pipeline (previously undocumented there — only `blueprint/SETUP.md` § 7g referenced it) and adds the missing step: once a second account config dir exists (`~/.claude2`, `~/.claude3`, …), its `commands/` (and `projects/`) must symlink to the primary's, or `/bb` and every other global command is invisible on that account.
- **`host-swap/README.md` — Fleet management section matches the synced behavior** — the attach view, auto-unhide, the two new reap guards, and the poll-based `/bb` close.
- **`blueprint/README.md` — Host tooling bullet lists the chat fleet** (`cc-ls`, `/bb`, `cc-reap`), previously omitted alongside the statusline/VSCode/`/swap` mentions.
