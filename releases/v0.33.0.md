# v0.33.0 — 2026-06-17

The command surface adopts a uniform nested-command shape (`/wave:build`, `/quality:*`, `/wave:{refine,review}`, `/p:*`, `/audit:*`, `/pcm:{update,release}`), the three Tier-A personas are restored to full depth, a generic file-protection gate ships, and `/chat:interrogate` joins the `/chat:` family.

## Added

- **Mechanics: `km-guard.sh` — a generic file-protection gate** (`templates/scripts/km-guard.sh` + a `PreToolUse(Edit|Write)` hook in `settings.json`). Blocks edits to a configurable `PROTECTED_PATHS` set unless a marker file is present (with a TTL), returning a deny message with instructions to open the gate. Adopters fence off their own sensitive files — prompts, schemas, secrets configs — from careless or unauthorized automated edits. Mechanism is domain-agnostic; protected paths are an install-time fill.
- **Tier A: `/chat:interrogate`** (`commands/chat/interrogate.md`) — interrogate a finished session to recover its REASONING (not just its conclusion) by discovering the predecessor and fork-resuming it to ask one read-only question. Joins the existing `/chat:` family.
- **Mechanics: `/p:360` ships as a portable command** (`commands/p/360.md`). It previously existed only in the optional Codex layer, so non-Codex installs had no 360°-sweep utility; it is now a first-class nested command for everyone.

## Changed

- **Mechanics: uniform nested-command structure.** What used to ship as standalone `templates/skills/p:*` skills are now nested COMMANDS, each registering via its frontmatter `name`: `quality/{prompt,doc}`, `wave/{refine,review,build}`, `p/{rnd,360}`, `audit/{code-hygiene,security}`, and the `pcm/{update,release}` bus (with `pcm/references/refresh.md`). A bare `/ns` command is a flat `commands/ns.md`; its colon-subcommands live in `commands/ns/{cmd}.md` (a dir with no `SKILL.md`). `templates/skills/` now holds only `sources.json` (the source-fetched skills `rr`/`ghostwriter`/`vision-factory`, never vendored).
- **Mechanics: `/build` → `/wave:build` everywhere.** The build command is nested under `wave/` (`commands/wave/build.md`, name `wave:build`); every reference across CLAUDE.md, the root agents, docs, the workflows, and the **Codex dual-runtime layer** (the `build.toml` wrapper → `wave-build.toml`, the `skills/build/` mirror → `skills/wave-build/`, every skill-mirror repointed to its new command path) was updated to match.
- **Tier A: all three personas restored to full depth** (`output-styles/{professor,jc,dr-house}.md`), genericized. The Professor regains "The relationship with the work," the "What NOT to do" guardrails, the worked Verdict examples, and the full voice-example set; JC regains its full trait set incl. the "don't break character for technical depth" rule; Dr. House regains its full worked examples. Personas are Tier-A — voice is load-bearing — and the templates had drifted thin.

## Migration

#### → For adopters with an installed blueprint:

- **Command structure (skills → nested commands).** `/pcm update` moves `p:quality:{prompt,doc}` → `commands/quality/{prompt,doc}.md`, `p:wave:{refine,review}` → `commands/wave/{refine,review}.md`, `p:rnd` → `commands/p/rnd.md`, `p:audit:{code-hygiene,security}` → `commands/audit/{code-hygiene,security}.md`, and the build command → `commands/wave/build.md`. If you customized any of these as standalone skills, re-apply onto the nested command file. Invocation changes: `Skill("p:quality:prompt")` → `Skill("quality:prompt")`, `/build` → `/wave:build`, `/p:wave:refine` → `/wave:refine`. Update any command-list contract/check script accordingly.
- **`/build` rename ripples into Codex.** If you run the optional Codex layer, the `/build` wrapper (`codex/agents/build.toml`) is renamed to `wave-build.toml` and the Codex skill-mirrors are repointed; re-pull the Codex layer wholesale rather than patching.
- **New `km-guard.sh` gate (opt-in).** `/pcm update` adds the script + the `PreToolUse(Edit|Write)` hook. It is a NO-OP until you fill `PROTECTED_PATHS`. Note: the gate's marker is bounded by a TTL; if you want it cleared on every turn, wire a Stop hook to clear `tmp/gate_marker` (the shipped Stop hook only runs notify).
- **`/chat:interrogate` + `/p:360`** — `/pcm update` adds `commands/chat/interrogate.md` and `commands/p/360.md`. Add them to any command-list contract/check script.
