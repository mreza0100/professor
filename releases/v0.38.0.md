# v0.38.0 тАФ 2026-06-26

A new feature: the three Tier A personas now ship in two selectable depths тАФ **full** (the rich, showcase version; the voice IS the value) and **compact** (a lean version that preserves the behavioral contract тАФ the Verdict rule, the sacred-ground red lines, the Analysis Protocol тАФ and each character's distinct voice, with the worked-example flavor and PhD prose trimmed). SETUP Q2 picks the depth; either way the install ends with the canonical bare `.claude/output-styles/{name}.md` filenames.

## Added

- **Tier A: `templates/output-styles/{professor,jc,dr-house}.md` тЖТ `.full.md` + `.compact.md`** тАФ each persona template splits into two depth variants. `.full.md` is the existing rich persona, byte-for-byte. `.compact.md` is a lean rewrite that keeps the in-character mandate, the Professor's Verdict rule (format + examples), the sacred-ground red lines, the terse Analysis Protocol, and each character's distinct voice тАФ and surfaces the PhD/disciplines roster as a single visible one-line list тАФ while trimming the worked-example flavor and PhD prose. The bare `{name}.md` filenames are retired from the templates (the install still writes them тАФ see Migration).

- **Docs: `SETUP.md` Q2 тАФ persona-depth choice** тАФ "Character name & voice" gains a full-vs-compact depth pick. full is the showcase default (the voice is the value); compact trims flavor + PhD detail every turn while preserving the behavioral contract and voice. Because the Professor is the session output style loaded on every main-loop turn, compact's per-turn savings land mostly there; JC and Dr. House are command overlays loaded only when `/jc` or `/pcm` runs. One install-wide choice covers all three personas.

- **Docs: `SETUP.md` Phase 2 тАФ output-style install step (3a)** тАФ the customization phase now explicitly copies the chosen variant `{name}.{depth}.md` тЖТ `.claude/output-styles/{name}.md`, so an install always ends with the canonical bare filenames; the `.full`/`.compact` suffix exists only in the blueprint templates.

- **Docs: `SETUP.md` manifest seed тАФ `persona_depth`** тАФ `.professor/manifest.json`'s interview seed gains a `persona_depth: "full" | "compact"` field (recorded in the Phase-2 manifest-writing step and the example manifest block), so `/pcm:update` can replay the depth choice.

## Migration

#### тЖТ For: existing installs are unaffected тАФ your single `.claude/output-styles/{name}.md` files keep working exactly as before (they correspond to the `full` depth). To switch a persona's depth, re-copy the desired variant from the blueprint templates тАФ `blueprint/templates/output-styles/{name}.full.md` or `{name}.compact.md` тЖТ `.claude/output-styles/{name}.md` тАФ or re-run SETUP Q2 and let the installer recopy all three. Add `"persona_depth": "full"` (or `"compact"`) to your `.professor/manifest.json` interview seed to record the choice for future updates.
