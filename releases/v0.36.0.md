# v0.36.0 ‚Äî 2026-06-22

Memory backup is rebuilt from the single-project model into **ONE vault, MANY subdirs**: one private git repo holds every project's Claude memory, each project in its own subdirectory keyed by the project folder's basename ‚Äî so the same project shares one subdir across every machine. A `SessionStart` hook auto-wires whatever project you open (pull ‚Üí symlink the memory dir into its subdir, migrating a pre-existing real dir in first); a `SessionEnd` hook pull-rebases then commits + pushes the whole vault (multi-writer-safe, self-healing). A one-time consolidator migrates existing per-project memory dirs into the vault. The vault path is configurable through one point ‚Äî the `{MEMORY_VAULT_DIR}` placeholder (default `$HOME/work/{MEMORY_VAULT_DIR}`) with a `CLAUDE_MEMORY_REPO` env override ‚Äî never hardcoded. **This SUPERSEDES the v0.12.0 single-project memory-backup model.**

## Added

- **Scripts: `scripts/cc-memory-wire.sh`** ‚Äî `SessionStart` hook. Pulls the vault, then symlinks the current project's `~/.claude/projects/<key>/memory` into `<vault>/<project-basename>/`, migrating a pre-existing real memory dir in non-destructively first. Idempotent. **Root-guard:** a memory dir already symlinked to the vault ROOT (a legacy single-project "main brain") is left untouched, never re-homed.
- **Scripts: `scripts/cc-memory-consolidate.sh`** ‚Äî one-time consolidator (run once per machine). Migrates every `~/work/<project>` memory dir into its vault subdir: copy ‚Üí VERIFY file-for-file ‚Üí only then swap for a symlink. Skips any dir already linked into the vault, including the legacy root brain.

## Changed

- **Scripts: `scripts/memory-sync.sh`** ‚Äî rewritten from single-project SessionEnd push to **whole-vault** sync: `git pull --rebase --autostash` (multi-writer-safe) ‚Üí commit ‚Üí self-healing push. Reads the configurable vault path; the commit+push logic is folded into the shipped script (no dependence on a `.sync.sh` living inside the user's data repo). Log moves to `~/.claude/memory-sync.log`.
- **Docs: `references/memory-backup.md`** ‚Äî rewritten for the new model: one vault with per-project subdirs, `SessionStart` auto-wire, root-guard, the consolidator, cross-machine sharing by project basename, multi-writer safety, the single `{MEMORY_VAULT_DIR}` / `CLAUDE_MEMORY_REPO` config point, and new-machine restore via a single clone.
- **Docs: `SETUP.md` (Phase 4)** ‚Äî rewritten to the new model: create ONE vault, copy the three scripts to user-level `~/.claude/scripts/`, wire the `SessionStart` + `SessionEnd` hooks into global `~/.claude/settings.json` (cross-project, fire once per session ‚Äî not duplicated per project), run the consolidator once.
- **Docs: `PLACEHOLDERS.md`** ‚Äî registered `{MEMORY_VAULT_DIR}` (Identity family) as the configurable vault directory name, with the `CLAUDE_MEMORY_REPO` env override.

## Migration

### For: adopters who NEVER enabled memory backup

Nothing to do ‚Äî still opt-in. To enable the new model, follow `SETUP.md` Phase 4. **`/pcm:update`: ask before installing ‚Äî opt-in capability.**

### For: adopters on the v0.12.0 single-project model

The old model symlinked one project's `memory` dir to a `<project>-memory` repo and pushed via a `.sync.sh` inside that repo. To move to the one-vault model:

1. **Keep your existing vault or create a new one.** Your old `<project>-memory` repo can become the vault ‚Äî its current root content stays as the legacy "main brain"; the **root-guard** in `cc-memory-wire.sh` and the consolidator both detect a memory dir symlinked to the vault root and leave it untouched, so nothing re-homes it.
2. **Install the three new scripts** to `~/.claude/scripts/` (`{cc-memory-wire,cc-memory-consolidate,memory-sync}.sh`), substituting `{MEMORY_VAULT_DIR}` with your vault's directory name (or set `CLAUDE_MEMORY_REPO`).
3. **Replace the old `SessionEnd` hook.** Remove the `sh $HOME/work/<project>-memory/.sync.sh` command from `~/.claude/settings.json` and add the new `SessionStart` (`cc-memory-wire.sh`) + `SessionEnd` (`memory-sync.sh`) hooks (one-liner in `SETUP.md` Phase 4 / `references/memory-backup.md` Tip 9). The `.sync.sh` inside your data repo is no longer used ‚Äî the commit+push logic now ships in `memory-sync.sh`.
4. **Run the consolidator once** to fold any other per-project memory dirs into the vault: `sh ~/.claude/scripts/cc-memory-consolidate.sh`. New projects wire automatically on first open.
