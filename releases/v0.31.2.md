# v0.31.2 — 2026-06-17

A `--no-sig` flag for chat: injection — an explicit way to drop the sender signature for operational injections, alongside the existing slash-command auto-skip.

## Added

- **Scripts + Docs: `commands/chat/chat.sh` (inject) + `inject.md` — `--no-sig` flag.** `chat.sh inject --no-sig {target} "{message}"` drops the sender-signature footer in every arm — the LIVE typed path, the long-message spill file (both its pointer and its content), and the RESUME transcript append — so operational injections (`/compact`, `/goal`, `/loop`, and the like) land clean, especially when driving another chat programmatically. The long-message file-cap still applies; only the signature is suppressed. The `/`-prefix auto-skip (v0.31.1) stays as the implicit default for literal slash commands — `--no-sig` extends clean delivery to any message.
