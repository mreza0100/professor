# v0.28.0 — 2026-06-13

## Added

- Tier A: `/chat:load` — resume an earlier chat from a pasted excerpt. `chat-load.sh` greps every past session in the registry for the excerpt (current session excluded, hit-count ranked), extracts that session's full visible conversation via the shared `transcript-extract.jq` to `tmp/chat-loads/{session-id}.md`, and the command reads it all and picks up where the old chat left off.
- Tier A: `/goal-definer` — compile a fuzzy super-goal into a sharp, self-contained prompt for a fresh session in the same repo. Interrogates the founder (AskUserQuestion, ≤2 rounds) → grounds in repo docs/epics → writes `tmp/prompt-saves/{slug}.md` (the file is the sole output; the prompt is never echoed into chat). Every generated prompt opens with a fixed Operating-mode block (build the end-state mental model first, plan in workflows but decide each step dynamically, backtrack on real dead ends) and, for long-running goals, a Continuity block that creates an epic, copies the prompt to `goal-prompt.md`, and consolidates via `/documenter epic`. Name avoids `/goal` (a Claude Code built-in).

## Changed

- Tier A: `/save` → `/chat:save` — rebuilt around script-dumped transcripts and namespaced under `chat/`. `save-transcript.sh` resolves the live session JSONL from `$CLAUDE_CONFIG_DIR` + `$CLAUDE_CODE_SESSION_ID` and appends the verbatim chat (visible text + tool-name trail) plus a git environment snapshot; the briefing header shrinks to a 5-section shell (Mission / State of work / Hidden context / Open questions / Next steps), so the model spends tokens only on what the transcript cannot show. Epic saves remain in `/documenter epic` — the canonical `/chat:save` carries no epic subcommand.

## Migration

#### → For adopters with an installed blueprint:

`/save` is renamed `/chat:save` and moves to `.claude/commands/chat/save.md`; `/pcm update` places the new command, the paired `/chat:load`, and `/goal-definer`, adds the three scripts (`save-transcript.sh`, `chat-load.sh`, `transcript-extract.jq` — they require `jq`), and removes the old `save.md`. Re-point any `/save` muscle memory to `/chat:save`. If your install registers commands in a contract/check script, add `chat/save`, `chat/load`, and `goal-definer` to that list.
