# v0.18.0 — 2026-06-07

The templates now honor the README's "any repo / any stack" promise: structure is a **roster** (1..N projects, single-project first-class), not the origin's fixed five-project monorepo. Release notes are also split into one file per version.

## Changed

- Mechanics: **roster model** — `PLACEHOLDERS.md` replaces the fixed `{BACKEND_PROJECT}/{FRONTEND_PROJECT}/…` role set with a project **roster** (`{PROJECT_ROSTER}` + per-entry `{project}`/`{PROJECT_ROLE}`/`{PROJECT_STACK}`/`{PROJECT_PKG_MGR}`/`{PROJECT_TEST_RUNNER}`/`{PROJECT_PORT}`) plus a materialization-expansion token family. Every template is roster-driven: `build.md`/`wave.md` per-project pipeline stages are single PATTERN blocks SETUP expands once per entry; `worktree.sh`/`dev.sh` iterate a `PROJECTS=()` array; `CLAUDE.md`, `SETUP.md`, the per-project agents, and the skills all read correctly at roster size 1. (breaking)
- Tier A: domain fully slotted — clinical/therapy/session/transcription literals → `{DOMAIN_NOUN}`/`{USER_NOUN}`/`{SUBJECT_NOUN}`/`{SESSION_NOUN}`/`{DOMAIN_SAFETY}`, so a non-clinical adopter reads clean. (safe-auto)
- Mechanics: **release notes split** — `CHANGELOG.md` is now a slim index; each release is its own file under `releases/vX.Y.Z.md` so adopters read one release at a time. `/blueprint release` writes a per-release file + an index line; `/pcm update` reads the per-release files between versions. (safe-auto)
- Tier A: `commands/blueprint.md` — `release` reconciles README/BLUEPRINT (cast + version) every release; `refresh` mines the source's structure INTO the generic roster pattern, never baking project count or role names into a template. (safe-auto)

## Fixed

- Docs: README/BLUEPRINT drift — cut vestigial `/council`, `/contentor`, `/qa`; clarified the source-fetched skills (`rr`, `p:360`, `ghostwriter`, `vision-factory`, `tokyo-night` theme); version examples made version-neutral. The universal "any repo / any stack" promise is kept — it is the contract the templates now honor. (safe-auto)
- Mechanics: `commands/pcm.md` mandatory-load referenced a stale skill name (`prompt-quality`) → `p:quality:prompt`. (safe-auto)

## Breaking

- The fixed five-project role tokens (`{BACKEND_PROJECT}`/`{FRONTEND_PROJECT}`/`{AI_PROJECT}`/`{INFRA_PROJECT}`/`{WEB_PROJECT}` as THE structure) are replaced by the roster model. Templates are now roster-driven PATTERN blocks materialized per entry at install.

## Migration

### For: all adopters

- This is a structural template change. On `/pcm update`, the roster-driven templates replace the fixed-five ones; re-run the SETUP materialization step so per-project PATTERN blocks expand to YOUR roster (a single-project install collapses to the repo root — no per-project subdirs, no cross-project steps). Your `.professor/drift.md` customizations are kept by the three-way merge.
- Release notes moved to `releases/` (one file per version); `CHANGELOG.md` is now the index.
