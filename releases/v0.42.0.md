# v0.42.0 — 2026-07-04

The `⚙ agent` row in `cc-ls` gains a **takeover/attach chooser**. Until now, Enter on a chat locked
by a live background agent could only *attach* to the running process — which keeps the account,
model, effort, and permission-mode the agent was BORN with, so swapping your primary account had no
effect when you reopened such a chat. The new `cc-agent-open.sh` adds the missing move: **take the
agent over** — stop it gracefully, then resume the same transcript fresh under the current primary
account.

## Added

- **`host-swap/cc-agent-open.sh` — the takeover/attach chooser for agent-locked chats.** A live
  background/forked session holds the session lock, so `claude --resume` refuses it, and *attaching*
  keeps the account/model/effort/permission-mode the agent was born with (`cc-swap`/`/swap` only
  affects new processes). The chooser, which `cc-ls` opens in the fresh tmux window, offers: **[t]
  takeover** (default when the agent is idle/blocked/done) — SIGTERM the agent gracefully (15 s
  grace, SIGKILL backstop), then resume the same transcript fresh **under the current primary
  account**; **[a] attach** (default when the agent is actively working — never kill in-flight work)
  — join the running process via `claude agents --cwd` under its owning account, keeping its original
  everything; **[q]** cancel. A stale `⚙` label (a registry entry with no live pid) resumes directly.
  N-account generic: the current primary maps `~/.claude-primary` number N → `~/.claudeN` (1 → the
  unset default `~/.claude`), and the agent registry is probed across the default account plus every
  `~/.claude[0-9]*` config dir on disk (the same convention `cc-reap`'s busy-scan uses).

## Changed

- **`host-swap/cc-ls.snippet.sh` — Enter on `⚙ agent` (and the resume failure-net) open the chooser.**
  The `A` branch and the `--resume`-refused fall-through now launch `~/.claude/bin/cc-agent-open.sh`
  instead of opening the `claude agents --cwd` attach view directly — so Enter can *take over* an idle
  agent, not only attach to it. No column or parse changes; a drop-in replacement over the v0.41.0
  snippet.

  #### → For: adopters who installed `host-swap/` — copy the new `cc-agent-open.sh` into `~/.claude/bin/` (`chmod +x`), then re-append `cc-ls.snippet.sh` to `~/.zshrc` (or re-source it). No account-table edits needed.

## Docs

- **`host-swap/README.md` — new "Agent mode — takeover vs attach" section.** Explains what background
  agents are (born-with account/model/effort/permission semantics), the takeover-vs-attach choice and
  its state-based defaults, and the N-account-generic account resolution; adds `cc-agent-open.sh` to
  the file table and the fleet-install step.
