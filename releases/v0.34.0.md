# v0.34.0 — 2026-06-22

`/chat:inject` is hardened to production: fully inline delivery (no file-spill), busy-target interruption, concurrent-sender serialization, and 🔖-label addressing. `/chat:capture` now grabs the full retained scrollback by default, and every `settings.json` hook command is anchored to `$CLAUDE_PROJECT_DIR` so hooks fire from any working directory — closing a latent safety gap where a `PreToolUse` guard ran from a subdirectory and silently did nothing.

## Added

- **Mechanics: `/chat:inject --force-now`** (`commands/chat/{chat.sh,inject.md}`). By default a message to a busy target queues behind its current turn; `--force-now` presses `Esc` to interrupt the running tool/flow so the target reads the message now, then delivers. It acts only on a live pane (ignored with a warning for a transcript/RESUME target) and only interrupts when the target is actually busy — an idle target is delivered to normally. When it interrupts, it appends a `⚠ FORCE-DELIVERED via Esc` marker so the recipient knows its work was cut off. Composes with `--no-sig`.
- **Mechanics: `/chat:inject` 🔖-label addressing.** Target a chat by its `/rename` name (the 🔖 in its status line) instead of its tmux session id — the script resolves a label to its session by scanning live panes the way `/chat:ls` does. Case-insensitive; an ambiguous label (two chats share it) errors and asks for the session id. Works everywhere a target is accepted, alongside `self` / tmux-session / session-id / transcript-path.
- **Mechanics: `/chat:inject` concurrent-sender lock + input-surface normalization.** When several chats inject into the **same** live pane at once, their keystrokes would interleave into one mangled turn. Each LIVE delivery now takes a per-target atomic `mkdir` lock (under `${TMPDIR}/chat-inject-locks/`, released on exit) so deliveries to one pane run one-at-a-time; a second sender waits up to `CHAT_INJECT_LOCK_TIMEOUT` then warns rather than colliding, and a lock whose owner died or was held too long is reclaimed automatically. Before typing, the script also exits the target's copy/scroll mode and dismisses a stuck Rewind menu (both silently swallow input), and uses a double-tap-Enter with positive submit confirmation against the input line so a swallowed first keypress can't leave a turn typed-but-unsent.

## Changed

- **Mechanics: `/chat:inject` fully inline delivery.** The long-message file-spill mechanism (a `📨 read this file` pointer for messages over a length cap) is removed — any message, however long, is typed inline. Sender identity is now **script-derived** (the sender's own tmux session as the reply handle + short session id + 🔖 label, read from the statusline); the `CHAT_INJECT_FROM_NAME` env var it replaces is no longer read. The footer is `— sid {id} · to reply: /chat:inject {handle} <message> · 🔖 {label}` (LIVE single-line; RESUME block).
- **Mechanics: `/chat:capture` full scrollback by default.** Captures the entire retained scrollback (`tmux capture-pane -S -`, bounded only by the target pane's `history-limit`) instead of the visible fold; the optional `[scrollback-lines]` depth argument is removed (no depth to guess, nothing amputated above the fold).

## Fixed

- **Mechanics: `settings.json` hook paths anchored to `$CLAUDE_PROJECT_DIR`.** All hook commands used repo-relative paths (`.claude/scripts/<name>.sh`). A hook command runs with CWD = the tool invocation's CWD, not the project root, so a session operating from a subdirectory with no own `.claude/` failed every hook (`/bin/sh: .claude/scripts/<name>.sh: No such file or directory`). Effect: the test-output filter never suppressed passing-test noise, `format-md` never ran, and — most seriously — **`km-guard`, a `PreToolUse(Edit|Write)` safety gate, was silently unenforced.** Anchoring every hook to `"$CLAUDE_PROJECT_DIR/.claude/scripts/<name>.sh"` (Claude Code's documented CWD-independent project-root env var) makes them fire from any directory.

## Migration

#### → For adopters with an installed blueprint:

- **`settings.json` hook paths.** `/pcm:update` re-applies the `$CLAUDE_PROJECT_DIR/` prefix to every hook command. If you customized `settings.json`, it surfaces as a both-changed conflict — re-apply the prefix to each `command` path. Hooks load at session start, so the fix takes effect on the **next session restart**, not hot.
- **`/chat:capture` argument drop.** The `[scrollback-lines]` argument is gone; drop the extra arg from any script that passed a line count — capture is always full-scrollback now.
- **`/chat:inject` identity.** `CHAT_INJECT_FROM_NAME` is no longer read — identity is derived from the sender's own tmux session + 🔖 label. Remove it from any wrapper; a leftover value is harmlessly ignored.
- **New optional env vars** (all defaulted, no action required): `CHAT_INJECT_LOCK_TIMEOUT` (30s), `CHAT_INJECT_LOCK_MAXHOLD` (60s), `CHAT_INJECT_LOCK_POLL` (0.1s), `CHAT_INJECT_ENTER_GAP` (0.15s), `CHAT_INJECT_ENTER_SETTLE` (0.4s), `CHAT_INJECT_ENTER_TRIES` (12), `CHAT_INJECT_INTR_TRIES` (8).
