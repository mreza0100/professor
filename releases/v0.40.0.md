# v0.40.0 — 2026-07-02

Two threads land together. The `chat:` family gains **teammate orchestration** — spawn a fork or a fresh chat beside you (or headless in the background), drive it by name, read its screen back without leaving your own — plus the cc-fleet tools (`cc-ls`, `/bb`, `cc-reap`) in `host-swap/`. And the **prompt-authoring discipline** is refactored: `/quality:prompt` becomes pure prompt-law, the file-shape skeletons move to `/pcm`, and a new `pcm-guard.sh` edit-gate protects `.claude/**` + `CLAUDE.md` the way `km-guard` protects knowledge files.

## Added

- **`/chat:branch [name]` — fork this chat into a side-by-side pane.** `split-window -h` runs a `--fork-session` resume into a fresh session id (the original transcript is never touched), pinned to this chat's model and named via `--name`. New `templates/commands/chat/branch.md` + a self-contained `branch)` case in `chat.sh`.
- **`/chat:new [--detach] [name]` — spawn a fresh teammate chat.** Default: a visible side-by-side pane (like `branch`, but a new empty chat, not a fork). `--detach`: a headless detached session on its own `cc-new-*` socket — off-screen, driven by name. Auto-named from this chat's prefix with the next free number (e.g. `RR` → `RR_1`, `RR_2` …). Model-invocable — the model can spawn its own teammates to do work while it orchestrates (the four-eye principle).
- **cc-fleet management tools → `host-swap/`.** `cc-ls.snippet.sh` (an fzf picker over every live + resumable chat), `cc-hide.sh` (`/bb`'s pane-aware engine), `cc-reap.sh` (reclaim RAM from the `cc-*` socket graveyard), and `bb.command.md` (the `/bb` command). Launcher-agnostic — they work with any setup that runs each chat in its own `tmux -L cc-*` socket. See `host-swap/README.md`.

  #### → For: cc-fleet tools are **opt-in host tooling** (like the rest of `host-swap/`). To adopt: `cp cc-hide.sh cc-reap.sh ~/.claude/bin/`, `cat cc-ls.snippet.sh >> ~/.zshrc` (needs `fzf`), `cp bb.command.md ~/.claude/commands/bb.md`. Skip if you don't run the cc-* socket fleet.

- **`/pcm` § Authoring conventions — the file-shape reference.** The frontmatter + file-skeleton templates for agents, commands, skills, and CLAUDE.md (plus two "well-shaped" worked examples) now live in `/pcm`, the command that actually authors `.claude/` files — relocated out of `/quality:prompt` so the prompt-law skill stays runtime-agnostic.
- **`/p:2opinion` — blind independent second opinion.** Spawns one fresh agent, blind to the current chat, to reach its own verdict on the problem from the primary sources, then contrasts it with yours. Productizes the context-isolation rule for adversarial review.
- **`pcm-guard.sh` — an edit-gate for framework files.** A `PreToolUse(Edit|Write)` hook that blocks edits to `.claude/**` and any `CLAUDE.md` unless `/pcm` is active, mirroring `km-guard` for knowledge files — so framework changes route through `/pcm` (which loads `/quality:prompt` and opens the gate).

  #### → For: copy `templates/scripts/pcm-guard.sh` (`chmod +x`) and add it to `settings.json` `PreToolUse(Edit|Write)` ahead of `km-guard.sh`. Skip both if you don't want the gate.

## Changed

- **`chat:inject` — pane-precise delivery + capture-proof.** Resolves a 🔖 name to its exact PANE (not the tmux session), so a backgrounded teammate — or a `branch`/`new` pane that shares its parent's session — receives an inject via `send-keys -t %N` with no `select-pane`: it never steals focus or breaks a zoom. Every live inject now returns the target's screen as **delivery proof**, so an orchestrator reads a headless teammate's state without attaching. `--then` follow-up steers to the same pane. Backward-compatible: single-pane chats behave exactly as before.
- **`/quality:prompt` — pure prompt-law, sharper.** The file-shape skeletons moved out to `/pcm § Authoring conventions` (above), leaving the skill to carry only the runtime-agnostic law. Added **Cut mode** (`quality:prompt cut <file>` — trim a file in place), **Compact aggressively** (the second pass under the cut test, with before→after transforms), and three anti-patterns: **#17** cross-document contract restatement, **#18** token-heavy formatting (HTML/XML tags + ASCII boxes → markdown), **#19** list-item definitions (`- term: gloss`, not bolded em-dash run-ons). The anti-patterns are now numbered.

  #### → For: if you hand-customized `/quality:prompt`'s "Structural conventions" section, that content now lives in `/pcm § Authoring conventions` — reconcile there, not in the skill.

- **`CLAUDE.md` § Model Selection — effort + nesting doctrine.** Added an **Effort** ladder (Max / XHigh / High / Medium / Low) and an **Agent-nesting** rule: complex work nests mixed tiers (a Sonnet fans out Haiku probes, then reasons over them), and a heavy context-flooding tool call dispatches a nested worker rather than running in the main loop.
- **`settings.json` — `pcm-guard.sh` wired into `PreToolUse(Edit|Write)`** ahead of `km-guard.sh`.
- **Professor Verdict format** gains an optional trailing segment for a question or steering request to the founder.
