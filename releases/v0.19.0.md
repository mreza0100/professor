# v0.19.0 — 2026-06-10

## Added

- Tier A: `/save` command — `/save {description}` dumps the session's complete working context (mission, verbatim founder instructions, state of work, files touched, decisions, gotchas, environment, open questions, the EXACT last 5 chat rounds verbatim, and next steps as the closing footer) to gitignored `tmp/prompt-saves/{name}.md` as a self-contained continuation briefing for a fresh chat, with a mandatory completeness pass. Invoke before `/compact` or when handing work across chats. Subcommand `/save epic {epic-name?}` routes the same dump into the epic's persistent context (`docs/epics/{name}/save-{topic}.md` + manifest update) so a fresh chat continues via "Load epic {name}".

  #### For adopters

  New file `templates/commands/save.md` → `.claude/commands/save.md`. `update`: auto-apply (new Tier A file, no placeholders).

## Changed

- Mechanics: archive flow — the documenter no longer moves pipeline dirs; gitter DOCS-COMMIT commits everything (including pipeline/wave files) into git history, then moves the dirs to gitignored `tmp/dev/archive/{builds,waves}/` and commits the removals, driven by a new `Archive:` directive from the orchestrator. Rolling counters, 10-item caps, eviction, and the zombie check are removed; `docs/` keeps no archive. `/wave` ends with one consolidated gitter DOCS-COMMIT covering all its build dirs plus the wave dir. (build, wave, documenter, gitter, mono-documenter, build-reference § 0a)

  #### For adopters

  After applying, migrate existing archives: move `docs/dev/builds/archive/*` and `docs/dev/waves/archive/*` (including `.counter` files) into `tmp/dev/archive/builds/` and `tmp/dev/archive/waves/`, remove the empty `archive/` dirs, and commit the deletions — the content stays in git history.

- Tier B: `p:refine` — every founder question (R1.5 discovery, R-POC approval, R2.5 relay, R3.5, R4 gate, P2) goes through the `AskUserQuestion` tool, preceded by a brief prose explanation; large sets batch into consecutive calls, ≤4 questions per call. **`update`: safe-auto — behavioral refinement, no structural change.**

- Mechanics: two-tier model policy — Opus for every real-work agent (root strategists pin the opus model id in frontmatter; all `build.md` child spawns pass `model: "opus"`; child agent frontmatter records `opus`), `sonnet` only for small jobs (gitter, mono-documenter, contentor-style auxiliaries); the agent-models reference is the authoritative tier map.

  #### For adopters

  Touches `build.md`, root agent frontmatter, and every per-project agent template. If you customized agent frontmatter, expect review-bucket conflicts — keep your local model pins where you diverged deliberately.

- Scripts: `notify.sh` — the Stop notification now identifies its origin: the title carries the project (git-root basename) and the body carries the short session id parsed from the hook's stdin JSON, so the originating chat is findable via `/resume`. **`update`: safe-auto.**

## Breaking

- Tier A: `/blueprint` → `p:blueprint` skill — the blueprint command is restructured into the `p:blueprint` skill with exactly two subcommands: `update` (the former `/pcm update` Update Protocol, moved out of `pcm.md`) and `release` (the former `/blueprint release`, with the refresh pass internalized at `references/refresh.md`). The standalone `export`, `refresh`, and `status` subcommands are folded away — refresh runs inside `release`; previews come from `update check`. `/pcm update` and `/pcm release` now route through the skill.
