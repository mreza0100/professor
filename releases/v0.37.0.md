# v0.37.0 — 2026-06-26

Fourteen changes ship in this minor release: two new workflows (wave-review, documenter-fanout), a new deferred-execution command (`/sleep`), a new code-hygiene Sweep Mode, two new `/wave:review` threads (Dead-code ripple, Test-data discipline), prompt-by-example wisdom, and a set of behavioral upgrades to `/jc:wave`, chat:, `/wave:refine`, the test stack, model-tiering, and `/pcm:release`.

## Added

- **Commands: `/sleep <30m|2h|1h30m|90s> <prompt>`** — deferred execution: persists the prompt to `tmp/sleep/<stamp>/task.md`, arms a background sleep, ends the turn, and on harness re-invoke runs the prompt verbatim. Self-indexing; `disable-model-invocation: true` (user-triggered only).

- **Workflows: `wave-review`** — `/wave:review` now runs as a Scout → Walk (one agent per thread, parallel) → Synthesize workflow, replacing the main-loop dispatcher fan-out. `/audit:code-hygiene` is merged into each walk (one read per file); the wave-level hygiene lens targets the integration delta, chiefly a repo-wide reuse-grep for helpers duplicated across sibling pipelines. Walker schema gains structured hygiene findings; synthesizer folds both functional and hygiene into the unified action-item list.

- **Workflows: `documenter-fanout`** — the serial documenter is replaced by a read-only scout that maps a consolidation's blast radius into disjoint doc scopes (one write-set each), then spawns one worker per scope to merge its slice in parallel. The build workflow inlines the same scout → fan-out pattern. The disjoint write-set partition is the race-free safety invariant.

- **Commands: `/audit:code-hygiene` — Sweep Mode** — new `sweep` scope removes confirmed-dead code and unused dependencies end-to-end behind QA: detect → adversarially prove deadness (zero consumers across the whole roster AND the dynamic-reference surfaces a static import-grep misses) → tier (confirmed-dead / uncertain / kept) → human approval gate → cut in a worktree, full QA, merge. Root `CLAUDE.md` "Surgical changes only" gains the carve-out: dead code and unused deps are removed deliberately through the sweep, never stripped inline during feature work.

#### → For: update your root `CLAUDE.md` "Surgical changes only" rule to add the dead-code sweep carve-out; the sweep's detect → prove → tier → approve → QA sequence is the only authorized path for removing dead code or unused dependencies.

- **Skills: `/wave:review` — Dead-code ripple thread** — a new review thread licensed to leave the wave's diff: from each symbol the diff removed, renamed, or dropped, the walker greps callers and importers repo-wide to catch code orphaned in unchanged files, traces one hop for cascades, and files each finding as a defect (`remove-if-dead`, or restore the forgotten wiring). Bounded — not a full-repo sweep.

- **Skills: `/wave:review` — Test-data discipline thread** — new blocking review thread scanning each wave's changed test and migration files against the rule "Tests own their data; the schema owns itself": no DDL or `.sql` in test code, rows created inline at scenario start, all schema and reference SQL in migration files (immutable system-reference-data baseline-seed exception), never read a migration file by name (introspect the live DB). New blocking gates: DDL-in-tests, seed/schema drift, migration-file coupling, migration-paper-over fixture.

#### → For: add the DDL-in-tests, seed/schema drift, migration-file coupling, and migration-paper-over blocking gates to your per-project QA agent protocols.

- **Skills: `/quality:prompt` + `/p:rnd` — prompt-by-example wisdom** — `/quality:prompt` gains "Teaching by example — when a stated rule keeps leaking": a rule the model keeps violating despite being stated is confusion, not disobedience; emphasis adds noise; a contrastive wrong → right example cures it; every "avoid X" example needs an "X is correct here" counterweight or the model over-suppresses X's legitimate uses. `/p:rnd` Phase 2 gains "Prompt RNDs — build the fix from ranked failures": collect failure modes → rank by frequency → cluster by shared confusion → place the fewest examples on the boundary the cluster crosses → confirm on a held-out case, citing `/quality:prompt`.

## Changed

- **Workflows: `/jc:wave` — wave-directory parity** — on wave start: names the wave, creates `docs/dev/waves/{wave-name}/`, copies the resolved spec to `manifest.md`, atomically resets the consumed root `wave.md` to a stub. Persists the review input and report in the wave dir; a new final step commits the wave dir via the git owner and archives it to `tmp/`. The consumed spec never lingers at root; `docs/` never accumulates stale wave files.

- **Mechanics: affected-first test ordering** — at every test gate, tests touched or added (plus directly affected ones) run first as a fast confirm; only on green does the full suite run once — never looped to chase a fix. Canonical rule in root `CLAUDE.md`; operational steps in `/jc` Step 4c, the per-project QA agents, and the build workflow (inherited by `/jc:wave` via its `/jc` reference).

#### → For: update your per-project QA agent protocols and build command to implement affected-first ordering — run touched/added/affected tests first, full suite once on green.

- **Scripts: `chat.sh` — tmux-socket-aware** — cross-chat ops (`ls` / `inject` / `capture` / resolvers) now scan every tmux socket under `${TMUX_TMPDIR:-/tmp}/tmux-$(id -u)/` and address targets by `(socket, session)` via `tmux -S`, instead of assuming the connected/default socket. Enables per-chat tmux-server isolation so one tmux server crash no longer kills the whole chat fleet. Backward-compatible (the default socket is just one of the scanned sockets); self-targeting subcommands untouched.

- **Skills: `/wave:refine` — shift-left reuse lens** — the R1 reconciliation note gains a "Reuse targets" field: apply the code-hygiene Duplication and Missed-Reuse discovery to each task at spec time, naming the existing helpers, components, hooks, and types to import rather than rebuild — catching reinvented helpers before any code is written.

- **Mechanics: model-tiering — detect/walk → Sonnet** — child planners retiered Opus → Sonnet (the build workflow's planner agent and per-project planner frontmatter): the planner is the find-half of a find → verify chain, where Sonnet wins on both speed and detection quality. The Opus mono-planner renders the routing verdict; Opus architects re-walk (judgment roles stay Opus). The documenter doc-scout retiered to Sonnet (read-only blast-radius mapper only).

#### → For: retune your per-project planner and doc-scout model pins from Opus to Sonnet; keep all judgment roles (mono-planner, architects, synthesizers) at Opus.

- **Commands: `/pcm:release` — update-gate (Step 0)** — release now refuses to run from a stale baseline: compares the local blueprint `VERSION` against the highest published tag and runs `/pcm:update` first if a tag is newer, only then releasing. Closes the collision where a repo lagging its own last publish recomputes an already-shipped version and collides on the tag push.

## Fixed

- **Scripts: `filter-test-output.sh` — subagent propagation** — root cause: `settings.json` hooks fire only for the main loop, not subagents, so the global `PostToolUse(Bash)` filter never reached QA agents or build workers. Fix: `filter-test-output.sh` is made dual-mode — the existing hook (main loop, unchanged) and a new `-p` pipe mode (raw stdin → failure-biased filter on stdout; retains coverage TOTAL/All-files lines). QA protocols, the build workflow, the build command, and root `CLAUDE.md` now pipe every test runner through `filter-test-output.sh -p`; `tail` / `head` / `grep` on test runner output are banned (typecheck / lint / build stay bare).

#### → For: update your per-project QA agent protocols and build command to pipe every test runner through `.claude/scripts/filter-test-output.sh -p`; explicitly ban `tail` / `head` / `grep` on test runner output in those protocols.
