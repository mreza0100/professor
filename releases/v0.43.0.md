# v0.43.0 — 2026-07-05

The wave engine grows up. The single `/wave:build` pipeline and `/wave` batch runner are replaced by a family: `/wave:refine` specifies, the new `/wave:schedule` sequences, `/wave:orchestrator` runs dual-chat trains in worktrees, `/wave:builder` implements (orchestrated or standalone), `/wave:live` batches on `main`, and `/wave:walker` verifies every merge. Around it: mechanical rate-limit plumbing (statusline harvest → sensor → prompt hook), guard scripts with session-keyed teeth, the model-selection doctrine folded inline into `CLAUDE.md`, and reference-card splits that cut the standing context of the most-spawned agents.

## Breaking

- **Mechanics: `/wave:build` → `/wave:builder`, `/wave` → `/wave:orchestrator`, `/wave:review` → `/wave:walker`, `/jc:wave` → `/wave:live`.** The wave family now lives entirely under `commands/wave/` — `templates/commands/wave.md` and `templates/commands/wave/build.md` are gone. `/wave:builder` is dual-mode: orchestrated (driven by `/wave:orchestrator` over the `/chat` protocol) or standalone (one full worktree pipeline, the old `/wave:build`).

  #### → For: delete `.claude/commands/wave.md`, `.claude/commands/wave/build.md`, and `.claude/commands/jc/wave.md`; adopt the new `commands/wave/{orchestrator,builder,refine,walker,live,schedule}.md`. Sweep your own docs/epics for the old names — every routing phrase changes (`/jc` any size on main; `/wave:live` batches; worktree pipelines are optional isolation, never required by size).

- **Mechanics: workflows — `wave-pipelines.js` retired, `wave-review.js` → `wave-walker.js`, `wave-build.js` rebuilt.** The group scheduler is gone (the orchestrator sequences trains itself); the walker is a new hybrid engine (thread walk + zero-token ledger rule-engine + judges + one final frontier judgment) with four modes: walk, VERIFY (claims panel), MANIFEST-VERIFY (spec hallucination gate), INVESTIGATE (goal-driven evidence-ledger crawl). `wave-build.js` drops the Plan/Architecture stages (~83% redundancy against a ZERO-GAP spec) and reorders GATE-1 after code review, so review fixes are re-gated by the full suite before merge.

  #### → For: replace `.claude/workflows/wave-review.js` with `wave-walker.js`, delete `wave-pipelines.js`, re-adopt `wave-build.js`. Invoke workflows by `scriptPath`, never by registered name — name-lookup serves a stale registry snapshot in a running session.

- **Mechanics: model doctrine — the `agent-models.md` registry is deleted.** `CLAUDE.md` § Model Selection now defines three tiers inline (frontier-judgment / spec-execution / collector), an effort ladder, the delegate-far-ahead rule, and a single frontier-declaration sentence (one edit to sunset a limited-run frontier model back to the default). Models are named in-place at each spawn site.

  #### → For: delete your `agent-models.md` (or equivalent registry) and adopt the new `templates/CLAUDE.md` § Model Selection; spawn sites carry their own tier aliases now.

- **Mechanics: `/qa` → `/qa:live`.** The root live-E2E command moves to `commands/qa/live.md`, out of the `qa-{project}` pipeline-gate namespace.

  #### → For: rename `.claude/commands/qa.md` → `.claude/commands/qa/live.md` if you adopted it; the per-project QA gates are unaffected.

## Added

- **Mechanics: `commands/wave/orchestrator.md` — the dual-chat wave orchestrator.** This chat orchestrates; a named live builder chat implements a ZERO-GAP wave task-by-task in one isolated worktree over the `/chat` protocol. Per-task verification with diff cards, milestone gates, GATE-1 pre-merge, then the boundary: GATE-2 ∥ walker post-merge on `main` while the next worktree is staged — with a founder-ratified OVERLAPPED-BOUNDARY option. Multi-area specs run as a wave TRAIN (one wave per code-impact area, each worktree cut only from verified main, both chats compacted between waves, train-wide `/goal` fired once). Includes the watchdog/sensor wake protocol, a six-way wake discriminator, steering-conscience retro entries, stop-the-train sequencing, account-limit GENTLE PAUSE, and context-discipline economics (builder chat defaults Sonnet-xhigh; explicit model/effort on every Workflow `agent()` call — verifiers silently inherit the caller's model otherwise).
- **Mechanics: `commands/wave/builder.md` — the wave builder.** Orchestrated mode (dispatch discipline with exact per-hand work-lists, incremental-ship for open-ended tasks, destructive-migration full-coupling checklist incl. raw-SQL grep + env-var scrub, BOUNDARY mode two-state machine, hard bans: no git of any kind, no full-suite runs in hands) and standalone mode (the full worktree pipeline: conditional design, develop, QA gates, merge, docs).
- **Mechanics: `commands/wave/schedule.md` — cross-spec wave scheduler, sole writer of root `wave.md`.** Collects QUEUED `/wave:refine` specs from `docs/dev/waves/queue/`, builds the cross-spec dependency + conflict graph (staleness probed mechanically via `git diff {Refined-sha}..main` ∩ file-plan), always gates conflicts on a founder ruling, and emits ONE consolidated dependency-ordered train. Division of labor: refine specifies, schedule sequences, the orchestrator executes.
- **Mechanics: `commands/wave/live.md` — batch tasks live on `main`.** Grouping and parallelism without worktrees: filesystem-safe parallel builds, end-of-wave `qa-{project}` agents, one `/documenter` + gitter commit, then the walker with inline remediation batched into boundary-lite `/jc` lanes.
- **Mechanics: `commands/wave/walker.md` + `workflows/wave-walker.js` — the post-wave verification walk.** A scout enumerates feature-flow/seam/invariant threads and schedules sensors over the API fields+gates the diff touched; a zero-token JS rule engine flags disconnects, encoding/casing mismatches, type drift, and auth-fence gaps; judges adjudicate only flagged anomalies; a diff-scoped security auditor rides the walk barrier; ONE frontier judgment rules the whole walk (may reinstate wrong kills) before the fold. Merge-SHA mode serves post-merge boundaries, branch mode manual pre-merge walks.
- **Mechanics: `commands/chat/goal.md` — compile an ambition into a runnable `/goal` and fire it at a live chat** (a teammate by tmux name, or this chat via self-inject). Goals are always inline text ≤4,000 chars; § Launch carries live-verified semantics (registers cross-pane, survives `/compact`, capture-readable ◎ mark, close by terminal condition). The orchestrator uses the non-interactive lane for train launches; the `epic` subcommand writes a resumable continuation prompt.
- **Mechanics: `commands/chat/self/compact.md` — author a focused `/compact` for this chat** (read the session first, write the hold + post-compact steer, self-inject).
- **Mechanics: `commands/pcm/context-meter.md` — context-consumption audit** across CLAUDE.md, agents, commands, skills, and MCP servers, ranked against size limits with prioritized savings.
- **Mechanics: `commands/audit/ai-output.md` + `workflows/audit-ai-output-sessions.js` — a third domain-hydrated audit shell:** three-angle faithfulness audit of AI-pipeline output read from the database against the source input and the pipeline's own code + prompts, with per-unit fan-out. Universal method ships; per-project channel/table/prompt knowledge is RR-hydrated at install.
- **Scripts: `wave-sensor.sh` — detached wake-economics sensor for the orchestrator.** Capture → regex-first classify (GOAL-DROPPED / IDLE / FINISHED-TASK / FROZEN / LONG-TURN / CTX / LIMITS / DEAD) → collector-tier model only on ambiguity → single inject escalation. Tick self-selects cadence by the inherited 1h-cache env (600s with, 270s without). The sensor classifies, never rules.
- **Scripts: `limits-hook.sh` — `UserPromptSubmit` hook that whispers account rate-limit gauges + reset countdowns into orchestrator turns only** (silent below 80%, on stale data, or in unregistered chats). Pairs with the statusline harvest below.
- **Scripts: `git-lock.sh` (cross-pipeline merge lock with stale-holder reclaim), `guard-stamp.sh` (PostToolUse-Read auto-stamp of the quality-load marker + session-scoped Stop cleanup), `checkpoint.sh` (per-worktree append-only audit trail), `drain-wait.sh` (generic wait-for-background-worker/queue-drain barrier with stall/timeout detection).**
- **Docs: reference-card trees** — `docs-commands/git/references/` (per-phase gitter cards: setup, merge, docs, push, wave, history), `docs-commands/documenter/references/` (per-scope merge cards + `doc-approval.md` write-rules/approval card + `sync-rules.md`), `docs-commands/jc/references/` (jc-core, debug-discipline). These carry the mechanics the slimmed agents now cite instead of restating.
- **Mechanics: `/sleep` rebuilt on the harness scheduler** — one-shot `CronCreate` (survives `/compact`, fires by enqueuing a turn, listable/cancelable) replaces the detached background `sleep` shell; sub-2-minute sleeps keep the background timer.
- **Tier A: `/wave:refine` legal-document hard fence.** Legal/compliance documents (DPIA, DPA, RoPA, privacy policy, consent docs, registers — anything of legal character) are founder-owned sacred ground: refine never edits one and a wave spec never carries a task/clause/file-plan routing that orders any agent to; code-only tasks + a paper-trail listing instead.

- **Mechanics: guards grow teeth — `pcm-guard.sh` / `km-guard.sh` rewritten** with dual session-keyed markers (from hook-stdin `session_id`), sliding expiry on every allowed edit (an active session never dies mid-batch; cross-session marker theft is structurally dead), and teaching deny messages carrying the exact unlock command. `guard-stamp.sh` makes the mandatory quality-skill load deterministic.

- **Mechanics: `statusline-command.sh`** — rate-limit gauges relabeled `5h-used:` / `7d-used:` (direction explicit), and a per-session harvest block writes the gauges to `/tmp/cc-rate-limits/acct-{n}.{session}.json` (account-resolved, window-alive gated, atomic, 1h self-reap) — the mechanical feed `wave-sensor.sh` LIMITS and `limits-hook.sh` consume (MAX across fresh files, erring toward pausing).
